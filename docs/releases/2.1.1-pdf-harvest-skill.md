# TaxaMask 2.1.1 PDF Harvest Skill Update

Release date: 2026-07-04

Branch: `main`

Tag: `v2.1.1`

This small release adds a bundled TaxaMask Agent skill for the missing PDF evidence stage 0: finding and harvesting legally accessible taxonomy PDFs before the existing PDF screening and figure/caption extraction workflow.

## Research Workflow Impact

The PDF evidence route no longer assumes that the researcher already has a folder of PDFs. When a user starts from a target taxon but has not collected the literature yet, the embedded Agent can now load `taxonomy-pdf-harvest` first, plan open metadata searches, download legally exposed PDF links, and preserve source-audit artifacts before TaxaMask screening begins.

For taxonomy work, this makes the origin of a PDF batch more reviewable. The downstream evidence database still records PDF, page, caption, nearby text, figure, and candidate provenance; this update adds the earlier acquisition layer that explains where the PDFs themselves came from.

## Included Changes

- Bundles `taxonomy-pdf-harvest` under `vendor/ant-code/config/skills/` so the embedded AntCode runtime can discover it as a default local skill.
- Adds skill metadata so AntCode can list and load the harvest skill.
- Updates the PDF evidence Ask Agent route to ask first whether the user already has PDFs or needs lawful open-access PDF harvest.
- Passes harvest-stage context from the PDF Evidence Tools button into Agent Center.
- Updates the Chinese user manual, README, and current Agent context document to describe PDF evidence stage 0.

## Output And Safety Boundary

The harvest skill writes source-audit artifacts such as `records.csv`, `doi_list.txt`, `summary.json`, `download_manifest.csv`, and `pdfs/`.

The skill uses open metadata and legally exposed PDF links only. It does not use Sci-Hub, LibGen, paywall bypasses, CAPTCHA bypasses, or logged-in scraping. Harvested PDFs and generated run outputs are local research artifacts and should not be committed to Git.
