# TaxaMask 设计历史索引

本文是公开版设计历史索引。它将历史设计稿整理为主题脉络，而不是逐字公开所有内部施工文档。

## 早期基础：项目、路径与持久化

早期设计重点是让 TaxaMask 从脚本式工具变成可迁移的本地项目：

- 高分辨率形态图像的标注需求定义。
- 相对路径和移动硬盘迁移。
- JSON 与 SQLite 的持久化取舍。
- PySide6 原生桌面工作台选型。

这些设计形成了后来的原则：项目应可离线使用，私有研究图像不需要上传服务器，保存结构应尽量可恢复。

## 2D 标注：Locator / SAM / Blink

随后设计转向 2D 标注主链路：

- Locator 与 SAM 两阶段标注。
- SAM 训练收敛和高精度分割修复。
- 显式框体监督和手动框导出。
- Locator 与 Segmenter 解耦。
- Blink 工作台入口、上下文记忆和小部位 refinement。

这些设计让 TaxaMask 从单纯画 polygon，发展为人工标注与模型辅助迭代结合的工作台。

## PDF Evidence

PDF 路线经历了从“文献筛选”到“证据链”的演化：

- 文献筛选通用化与多方案管理。
- 大规模 PDF 筛选运行加固。
- 三视图 Figure 提取。
- 真实多模态复核护栏。
- 图文提取与多模态 profile。

最终定位是：PDF 是 evidence/provenance 和候选材料来源，而不是自动训练真值来源。

## 多物种与 profile

TaxaMask 曾经探索更广的多物种适配，但当前公开定位更保守：

- 先稳定蚂蚁分类学工作流。
- profile 可让筛选、图文复核和模型配置更灵活。
- 不把未验证的跨类群能力作为默认产品承诺。

这也是 README 中强调“AntScan ant CT oriented validation”的原因。

## TIF/CT 与 AntScan

TIF/CT 路线来自内部形态研究需求：

- TIF 项目结构。
- AMIRA 只读导入。
- TIF 后端契约。
- GPU 体预览。
- 部位提取和右侧栏信息架构。
- Local Axis Reslice。
- TIF 训练、预测和预测结果导入。

这条路线把 TaxaMask 从外部形态图像扩展到内部三维形态，但仍保持人工复核优先。

## SQLite 迁移

随着项目规模增大，单个大 JSON 的风险变得明显：

- 保存慢。
- 崩溃时可能写成半截。
- 批处理越多，整文件重写越危险。

SQLite 迁移设计的核心不是“更工程化”，而是让大型研究项目可以增量保存、事务回滚和完整性检查。

## Agent Center

Agent Center 设计来自两个现实问题：

- TaxaMask 功能多，普通用户不应被迫读完整源码或长手册。
- 项目快速迭代时，需要一个本地、可审计、可确认的修复入口。

公开版设计只保留架构和安全原则，不公开本地 Agent 施工记录。

## 当前公开文档策略

历史文档的公开原则：

- 公开设计目标、架构边界、数据安全和验收摘要。
- 不公开本地路径、临时 handoff、一次性排查记录和私有研究输出。
- 旧设计如果与当前实现不一致，应通过索引说明其历史价值，而不是让用户误以为它是当前行为。
