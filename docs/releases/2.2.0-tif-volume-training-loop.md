# TaxaMask 2.2.0 TIF Volume Annotation, Training, and Prediction Loop

Release date: 2026-07-07

Branch: `main`

Tag: `v2.2.0`

TaxaMask 2.2.0 is a medium release that turns the TIF/CT workbench from a volume viewing, part extraction, and local-axis reslice workspace into a fuller reviewed volume-segmentation loop. The new route is still more experimental than the 2D/STL morphology workflow, but it now has the core pieces needed for practical internal-structure annotation, dataset preparation, backend training, trained-model reuse, prediction import, and human review.

The first validated use case remains AntScan-style ant CT data and internal morphology. The implementation avoids hard-coding the workflow to brain-only labels, so the same label-schema and backend structure can be reused for other parts or other volume-segmentation targets as users validate them.

## Research Workflow Impact

Before this release, the TIF/CT route could import stacks, inspect volumes, cut part volumes, generate part masks, and export local-axis reslices. Version 2.2.0 adds the missing reviewed training loop:

1. Define or import a shared region label schema.
2. Bind that schema to a part or local-axis reslice.
3. Select a label before painting.
4. Paint, erase, copy, or interpolate labels on TIF slices.
5. Save the editable label layer.
6. Explicitly accept reviewed labels as `manual_truth`.
7. Let TaxaMask collect all train-ready part/reslice samples across the project.
8. Prepare backend datasets and train through the TIF backend contract.
9. Register trained model manifests in the TIF model library.
10. Reuse an existing model for later prediction instead of retraining every time.
11. Import predictions as editable review layers with raw prediction backups.
12. Review and, if appropriate, accept corrected predictions as new training truth.

For research work, the most important boundary is preserved: model output and temporary edits do not become training truth automatically. A researcher must still review and promote data to `manual_truth`.

## Major Changes

### TIF Label Schemas

- Adds a TIF region label schema workflow for volume segmentation labels.
- Supports schema creation, import, export, saving, row editing, color swatches, and binding to the current part/reslice.
- Clarifies the difference between specimen material maps, region label schemas, current labels, and part group tags.
- Keeps label IDs and colors stable so multiple specimens can share the same numeric label meaning.
- Updates exported schema wording from brain-specific language to generic volume-segmentation language.

### Annotation And Review

- Renames and clarifies the user-facing TIF route as annotation and training rather than brain-only training.
- Makes the expected annotation order explicit: bind schema, choose label, then choose brush/eraser/fill tools.
- Keeps full-volume edits in `working_edit` and part/reslice review edits in `editable_ai_result`.
- Adds safer save-before-backend behavior: if labels are dirty or auto-save is still running, backend runs wait for saving or ask the user what to do.
- Keeps `raw_ai_prediction_backup` as read-only provenance for imported predictions.
- Prevents `editable_ai_result` and other draft layers from being used directly as training labels.

### Train-Ready Sample Collection

- `Prepare dataset` and `Train backend` now use a project-wide sample rule.
- TaxaMask first collects all train-ready part/reslice samples with reviewed `manual_truth`.
- If no train-ready part/reslice samples exist, it falls back to train-ready top-level specimen volumes.
- Train-ready checks verify image presence, label presence, shape compatibility, label schema/material consistency, and explicit human acceptance.
- One-sample `Prepare dataset` is useful for export-layout inspection; real nnU-Net v2 training requires at least two exported samples.

### nnU-Net v2 Backend Integration

- Adds and refines the bundled `taxamask_tif_nnunet_v2_backend` adapter.
- Exports nnU-Net v2 style raw datasets, NIfTI labels, dataset manifests, and label-ID mappings.
- Calls `nnUNetv2_plan_and_preprocess`, `nnUNetv2_train`, and `nnUNetv2_predict` from the configured backend Python environment.
- Writes TaxaMask model manifests after successful training.
- Remaps compact nnU-Net labels back to TaxaMask label-schema IDs before prediction import.
- Renames the default dataset name from the earlier brain-region wording to `TaxaMaskTifVolumeSegmentation`.
- Keeps the TIF backend contract backend-neutral so users or Agents can replace the command fields with MONAI, a custom 3D U-Net, or another volume-segmentation script.

### Trained Model Library

- Adds TIF model-library management for trained volume-segmentation models.
- Registers model manifest, run/result paths, sample counts, label schema IDs, usability metadata, and notes.
- Adds a `Use selected model` handoff so users can select an existing trained model for prediction.
- Adds note saving and registration-only deletion. Deleting a model record removes the project index entry, not the model weights or run folder on disk.

### Prediction Import And Review

- Adds batch prediction target selection for part/reslice targets.
- Supports top-level volume prediction fallback when operating in a full-volume context.
- Imports part/reslice predictions into `editable_ai_result` plus `raw_ai_prediction_backup`.
- Imports top-level predictions into pending-review `working_edit` plus `raw_ai_prediction_backup`, while preserving legacy `model_draft` audit compatibility.
- Adds shape and provenance checks around prediction import.
- Keeps prediction results out of `manual_truth` until the researcher explicitly reviews and accepts them.

### UI And Responsiveness

- Moves long TIF backend actions into background runs with progress/status/log panels.
- Adds clearer backend run state, elapsed time, run folder, result JSON, and log-opening controls.
- Reduces duplicate failure messages in the right-side panel.
- Improves save status and delayed-start behavior when a user starts prepare/train/predict while labels are still being saved.
- Clarifies color selection and label-schema help text.
- Removes brain-only wording from result comparison and label-table surfaces where the workflow is now generic.
- Keeps part group tags separate from annotation labels; group tags only organize prediction or review batches.

### Agent Context And Documentation

- Updates Ask Agent context for TIF with label schema, train-ready sample counts, selected training scope, backend command presence, run folders, result JSON, selected model manifest, and recent logs.
- Updates the detailed LLM handoff context so embedded agents know the current TIF architecture and data boundaries.
- Updates README and README_zh to mention the new TIF annotation, training, and prediction loop.
- Updates the Chinese user manual with a step-by-step TIF workflow from label schema to training, prediction, and review.
- Updates the public TIF backend contract to describe part/reslice and top-level prediction import behavior.

## Data And Safety Boundary

This release does not include user TIF/CT data, AMIRA data, PDFs, project SQLite databases, model weights, nnU-Net outputs, run folders, API keys, or local runtime settings.

Generated artifacts such as `TaxaMask_outputs/`, local TIF projects, SQLite databases, `.ome.zarr` volumes, NIfTI files, logs, and trained model files remain local research outputs and should not be committed to Git.

## Operational Notes

- The TIF/CT route remains less broadly validated than the 2D/STL morphology workflow.
- The first mature validation target is ant CT internal morphology. The route is no longer worded as brain-only, but broader taxa/body-part use still needs user validation.
- The configured backend Python environment must actually contain nnU-Net v2 commands. TaxaMask can prepare the contract and dataset, but it cannot train through nnU-Net if `nnUNetv2_plan_and_preprocess`, `nnUNetv2_train`, or `nnUNetv2_predict` are missing.
- For real nnU-Net training, prepare at least two reviewed train-ready samples.
- If a project already has old top-level prediction drafts, they remain audit records. New part/reslice prediction review uses `editable_ai_result` plus `raw_ai_prediction_backup`.
