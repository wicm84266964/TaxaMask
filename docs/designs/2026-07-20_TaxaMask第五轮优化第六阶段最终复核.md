分支：`codex/experimental-research-traceability`

## 结论

M5 与第五轮总体验收通过。训练可追溯、文件异常善后、推理阶段诊断、Blender 5.0 STL、Local Axis 高风险优先复核、三平台轻量 CI、数据安全检查和文档同步均已落地。SQLite 仍是项目唯一权威状态，未验证样本仍不能进入训练。

## 研究工作流变化

- 训练记录：项目 SQLite 保存实际事实；“训练结果”继续打开报告和模型产物，研究者用备注区分目的和重要性，不增加正式/临时两套训练。
- 文件不一致：训练停止并打开恢复界面，列出登记值/当前值，引导重新复验、同哈希重定位、恢复备份、登记新版本或导出脱敏诊断。
- 推理排错：现有 `TaxaMask_outputs/runtime_logs/` 可区分 Locator、路由和 SAM 等阶段，不记录原图、完整 mask、密钥或私人绝对路径。
- 网格：默认输出到项目 `exports/meshes/`，原始 STL 可用于测量，平滑 STL 仅作展示；中断记录可复验、重试或按记录安全清理。
- 复核：高风险排序解释原因但不代表错误概率；只接受用户明确选择，批量导出只处理已接受项，TIF 批量通过只处理选择集合。

## 剩余边界与回退

- 本阶段没有重跑真实 GPU 体渲染或新的真实 TIF 数据验收；相关自动契约通过，Blender 真实验收沿用 M3 记录。
- 高风险排序的受控合成试验只验证排序机制，不代表真实蚂蚁样本一定获得相同收益；取得真实复核数据前不扩展为错误概率或主动学习。
- 本地无法代替 GitHub 执行三平台远程任务；本分支按用户要求不添加 remote、不 push。
- M0 至 M5 各自独立本地提交。需要回退时按阶段提交处理，不删除项目数据库、运行产物或研究数据。
