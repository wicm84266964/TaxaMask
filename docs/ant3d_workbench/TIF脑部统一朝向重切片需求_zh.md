# TIF 脑部统一朝向重切片需求

> 日期：2026-05-29  
> 状态：需求文档，尚未实现  
> 目标用户：研究蚂蚁内部结构、脑部结构和后续 AI 训练数据准备的研究者

## 1. 背景

AntScan / micro-CT / 连续 TIF 体数据中的每个蚂蚁个体，起始摆放位置、头部角度、身体姿态和脑部朝向都可能不同。直接按原始 Z/Y/X 切片浏览时，一只个体的脑可能是完整横切，另一只个体可能是斜切或只切到局部。

用户希望把不同 specimen 的脑部统一到同一解剖学方向后重新切片，使训练和结构分析看到的是可比较的脑部 volume，而不是受扫描摆放角度影响的原始切片。

## 2. 研究目标

将每只蚂蚁的头部/脑部体数据重采样为统一方向：

- 头顶轴线，也就是 dorsal / head-top 方向，统一为垂直地面向上。
- 前后轴，也就是 anterior-posterior 方向，统一到同一个方向，避免绕头顶轴随机旋转。
- 裁剪出只包含脑部或脑部附近 ROI 的新 volume，尽量不包含胸部、口器、触角基部等无关结构。
- 输出可用于人工复核、结构测量或 AI 训练的标准化脑部切片堆栈。

## 3. 为什么不能只靠当前 3D 预览截图

当前 3D 体预览解决的是“看清楚”和“找角度”，它是屏幕渲染结果，不是新的训练数据。

脑部统一朝向重切片需要对原始 3D voxel 做明确的几何重采样：

- 使用原始灰度体数据，而不是屏幕像素。
- 保留或记录 `spacing_zyx`，避免比例变形。
- 图像体和标签体采用不同插值策略。
- 输出新的 volume sidecar / TIF / NIfTI 等数据，而不是单张截图。

## 4. 必需的方向约束

只指定“头顶方向向上”还不够。因为体数据仍然可以绕这根轴旋转，导致左右或前后方向不一致。

第一版需要至少记录：

| 标定项 | 用途 |
|---|---|
| 脑部或头部中心点 | 确定重切片和裁剪 ROI 的中心 |
| dorsal / 头顶方向 | 决定统一的上方向 |
| anterior-posterior / 前后方向 | 消除绕头顶轴旋转的不确定性 |
| ROI 尺寸 | 决定输出脑部 volume 的范围 |

后续可选增加左右方向、左右脑对称轴或头壳边界点，提高标准化稳定性。

## 5. 建议工作流

第一版建议做手动标定 + 自动重采样：

1. 用户在 TIF 工作台打开 specimen。
2. 通过 3D 体预览旋转、放大、近端剖切，找到脑区大致位置。
3. 进入“脑部重切片标定”工具。
4. 用户标定脑/头部中心、头顶方向和前后方向。
5. 用户设置脑部 ROI 尺寸，例如宽、高、深，单位优先使用真实 spacing。
6. 程序生成标准化预览切片。
7. 用户确认后，程序导出标准化脑部 image volume。
8. 如果当前 specimen 有标签层，程序可同步导出标准化 label volume。

## 6. 输出数据

建议输出到 specimen 目录下的新派生数据层，例如：

```text
specimens/<specimen_id>/derived/brain_reslice/
  image.ome.zarr/
  labels_manual_truth.ome.zarr/       可选
  labels_working_edit.ome.zarr/       可选
  brain_reslice_params.json
  preview_report.json
```

`brain_reslice_params.json` 至少记录：

- source specimen ID
- source image sidecar path
- source label sidecar path，若存在
- source shape 和 spacing
- 标定点坐标
- 输出 shape 和 spacing
- 旋转矩阵或 affine matrix
- ROI 中心和尺寸
- 图像插值方式
- 标签插值方式
- 创建时间和软件版本

## 7. 插值规则

图像体：

- 默认使用线性插值。
- 对特别细的内部结构，可提供三次插值选项。
- 不应改变原始灰度语义，必要时保留 dtype 或明确记录归一化策略。

标签体：

- 必须使用最近邻插值。
- 不能用线性或三次插值，因为 material ID 是类别，不是连续灰度。
- 输出后需要检查 material map 是否仍然完整。

## 8. UI 需求

TIF 工作台可新增一个“脑部重切片”入口：

- 入口位置：3D 体预览控制区或 TIF 工具栏。
- 第一阶段功能：
  - 选择/新增脑部重切片方案。
  - 标定中心点。
  - 标定头顶方向。
  - 标定前后方向。
  - 设置 ROI 尺寸。
  - 生成预览。
  - 导出标准化 volume。
- 预览应显示：
  - 标准化后的 Z/Y/X 切片。
  - 当前输出 shape、spacing、ROI 尺寸。
  - 是否包含 label volume。

## 9. 数据安全边界

- 重切片输出是派生数据，不覆盖原始 TIF sidecar。
- 不自动覆盖 `manual_truth`。
- 如果导出重切后的标签，它应作为新的派生标签或训练导出，不应静默替换原始人工真值。
- 所有标定参数必须保存，保证后续可以复核同一 specimen 的标准化过程。

## 10. 验收标准

第一版完成后，应至少满足：

- 同一 specimen 使用同一参数重复导出，结果一致。
- 输出 image volume 和 label volume shape 对齐。
- label volume 不出现原 material map 之外的新 ID。
- spacing 不丢失，导出报告可追踪源数据。
- 用户可以用标准化切片看到完整脑部 ROI，而不是仍混入大量非脑部结构。
- 至少有一个小体数据单元测试验证 affine/reslice 方向和 label 最近邻规则。

## 11. 后续自动化方向

当手动标定流程稳定后，可以考虑：

- 用头壳/脑区粗分割自动估计中心和方向。
- 用多个 specimen 的标定参数训练方向预测模型。
- 支持批量应用同一物种或同一扫描批次的初始参数。
- 将标准化脑部 volume 直接接入后续 AI 训练导出流程。
