# 开发版整理提交说明 2026-06-15

本说明对应一次开发版本地提交，用来把当前仓库里“适合继续留在开发版主线”的改动单独归账。

## 本次提交保留的内容

1. TIF 工作流入口保护
   - 程序新增 `TAXAMASK_ENABLE_TIF_WORKFLOW` 环境变量开关。
   - 默认隐藏实验性的 TIF 工作流入口，避免公开版或普通使用场景误进入未完全稳定的体数据链路。
   - 本机开发时设置 `TAXAMASK_ENABLE_TIF_WORKFLOW=1` 即可重新显示 TIF 入口。

2. TIF 工作台调试路径显示
   - TIF 工作台右侧栏新增 `Specimen 状态` 区域。
   - 默认只显示 specimen 状态、是否可训练、shape、dtype、spacing、modality。
   - 勾选 `显示调试路径` 后才展开原始 TIF、working volume、label、material map、import report 等路径。
   - 这样日常观察体数据时界面更干净，排查导入和保存问题时仍能快速核对文件位置。

3. 启动脚本去本机化
   - Windows 启动脚本不再只依赖固定的 `antsleap` Conda 路径。
   - 会优先识别本地虚拟环境、`taxamask` 环境、旧的 `antsleap` 环境，以及显式设置的 `TAXAMASK_PYTHON_EXE` / `TAXAMASK_PYTHONW_EXE`。
   - 这有利于开发版在不同机器和不同环境名称下继续运行。

4. PDF 新种复查逻辑修正
   - 移除了 `test_new_species` 文件名直接判定为新种的测试捷径。
   - 新增测试确认 LLM 复查必须依据模型返回内容，而不是依据测试文件名。
   - 这对真实文献筛选很重要，避免文件命名影响候选文献可信度。

5. 研究数据与运行产物保护
   - `.gitignore` 增加 TIF、STL、体数据格式、模型权重、训练输出、私有文档等忽略规则。
   - 目标是防止原始研究数据、大体积体数据、模型文件、本地配置和运行产物被误提交。

6. 嵌入式 Agent 配置去本机化
   - 移除部分本机绝对路径和本地网关地址。
   - Agent 说明改为使用当前仓库根目录和相对技能路径。
   - 这样后续开发版迁移或公开版挑选功能时，不会夹带个人机器路径。

## 后续迁移原则

开发版继续作为内部研发主线，优先承载 TIF、部位提取、颜色板、三维渲染优化、PDF 筛选可信度等实验和研究功能。

公开版作为发布镜像，只在功能稳定后按“完整功能包”迁移，例如：

- TIF 入口开关
- TIF 调试路径显示
- 渲染颜色板
- 部位提取与单独展示链路

不要每天机械同步两个仓库。这样可以减少冲突，也能避免把未成熟功能、私有路径或大体积研究数据带入公开版。
