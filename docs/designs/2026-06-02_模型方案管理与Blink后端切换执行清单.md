## 执行状态

2026-06-02 同日落地状态：阶段 0-12 已完成第一版实现与复核。下面原清单保留为任务拆解和审计记录，不再表示“尚未开始”。

已落地内容包括：

- 项目级 `model_profiles` 数据结构、旧项目迁移和 active profile 切换；
- route 专家显式记录 `expert_backend`、manifest、input size 和 backend params；
- `2D/STL Model Settings` 重组为 `Profile / Parent-part annotation / Child-part annotation / Inference / Advanced Extensions`；其中高影响的父部位/子部位模型来源切换集中到 `Advanced Extensions / 高级拓展`，父部位和子部位页保留摘要与日常训练参数；
- ViT-B Blink manifest、热力图 Blink dataset/trainer/inference、外部 Blink `predict_child` 契约；

- README、用户手册、LLM 上下文和 Agent 记忆同步。

2026-06-03 收尾校正：

- 根据使用者视角，`External Backend / 外部后端` 不再作为独立主称呼，统一改为 `Advanced Extensions / 高级拓展`。
- 父部位自定义脚本和子部位自定义 Blink 脚本的主要配置入口集中到高级拓展；`Parent-part annotation` 和 `Child-part annotation` 页只展示当前来源摘要并保留常规训练参数，避免用户误以为需要在三个地方分别切换。
- 删除方案已补确认语义：删除只移除当前项目 JSON 中的方案配置，不删除权重、外部脚本或 route expert 绑定。

## 执行目标

把 2D/STL 模型设置从“散落参数 + route 只认 ViT-B Blink 专家”升级为“可保存、可切换、可审计的模型方案”。第一版要先保证现有内置热力图 Locator + SAM + ViT-B Blink 流程不退化，然后逐步加入热力图 Blink 和外部 Blink 后端。

这不是一次只改 UI 的任务。它同时涉及：

- 项目 JSON 数据结构
- 模型设置界面
- route 专家记录
- 训练产物 manifest
- 父部位推理链路
- 子部位推理链路
- 外部后端契约
- 测试和文档
