# Blink 工作台合并设计方案

日期：2026-05-22

状态：已进入代码实施；主工作台合并版已完成第一轮实现与自动化复核。

实施摘要（2026-05-22）：

- 主标注工作台已新增“父子精修 / Blink”区域，显示当前父子路由、父级框状态和路由专家状态。
- 主工作台已接入标注框、收缩松框、自动标注子部位、自动收缩和训练当前子部位专家入口。
- 主工作台已加入父级上下文下拉框，用户可在多个父级候选之间手动指定，并写入项目记忆。
- 父级框默认支持固定长宽比，并可在 2D/STL 模型设置中调整；主工作台可临时解除比例锁定。
- 独立 Blink Workbench 已从日常 2D/STL tab 中隐藏，旧 Blink widget 和旧入口函数仍保留为兼容回退。
- 当前实现保持 TIF 工作台、PDF evidence skill 和既有 2D/STL 全图 AI 流程不变。

本文件记录 TaxaMask 2D/STL 标注工作台与 Blink 工作台合并前的产品和交互设计。合并前代码基线已在本地 git 标记为：

```text
pre-blink-workbench-merge
```

## 背景

当前 TaxaMask 的真实高频标注流程通常发生在 2D/STL 标注工作台：

```text
选择图像
-> 选择部位
-> 绘制 polygon 或 box
-> 运行自动标注
-> 人工修正
-> 保存 / 训练
```

Blink 工作台最初用于展示和验证父子关系下的小部位精修流程。当前进入 Blink 的路径是：

```text
在标注工作台选择图像和子部位
-> 依赖父部位 manual/auto box 作为 ROI
-> 弹出 Blink 入口选择
-> 进入独立 Blink 工作台
-> 在局部坐标中编辑
-> Apply to Global 映射回全局项目标签
```

这个流程在算法验证阶段清晰，但在真实标注中存在重复操作：

- 图像和部位已经在主工作台选定，却还要进入另一个工作台；
- Blink 工作台重复了部位列表、日志、模型区和部分绘制工具；
- 小部位标注本身也可以在高分辨率主图中完成；
- 用户需要理解局部坐标和 Apply to Global，增加认知成本；
- 频繁跳转会明显降低高通量标注速度。

因此，本次设计目标不是把 Blink 工作台整体塞进主界面，而是将 Blink 降级为主标注工作台内部的“父子关系精修引擎”。

## 目标

合并后的 2D/STL 主标注工作台应成为唯一日常标注入口：

```text
主标注工作台
-> 父部位固定比例框
-> 子部位自由标注 / 收缩松框
-> 父子路由专家自动标注
-> 自动收缩轨迹生成
-> 当前子部位专家训练
```

用户不再需要进入独立 Blink Workbench 才能完成小部位精修。

## 非目标

本次合并不应做成一次高风险大重构：

- 不立即删除 Blink 后端算法、训练线程、专家文件结构和路由 manifest；
- 不把路由专家完整管理界面搬进主工作台；
- 不改变 TIF 工作台；
- 不改变 PDF evidence skill；
- 不改变主项目 JSON 的基本标签结构，除非为了明确框角色新增小字段；
- 不取消设置中的父子路由专家配置。

## 核心概念

### 父部位

父部位是大而稳定、适合作为子部位上下文 ROI 的结构。

默认父部位来源：

```text
设置中的主定位器部位 / locator scope
```

对蚂蚁模板而言，默认父部位通常包括：

```text
Head
Mesosoma
Gaster
```

但父部位不应硬编码为这三个。不同项目可以在设置中调整主定位器部位，因此主定位器部位应作为父部位候选池。

### 子部位

不在父部位候选池中的部位默认视为子部位。

例如：

```text
Mandible
Eye
Antenna
Petiole
```

子部位需要绑定一个父级上下文，优先级如下：

1. 项目中记忆过的 `child -> parent` 关系；
2. 右侧部位树中的层级关系；
3. 主定位器部位候选池中的可用父部位；
4. 用户手动指定。

### 父级框

父级框不需要成为一种额外的新框。

父级框就是用户在主标注工作台中选中父部位后，用现有框选工具绘制的该父部位 box。

例如：

```text
选中 Head
-> 使用框选工具画 Head box
-> 这个 box 同时是 Head 的正式定位框，也是 Mandible / Eye 等子部位的父级上下文 ROI
```

### 子部位框

用户选中子部位后绘制的 box 是子部位标注框。它不应继承父部位的固定长宽比。

子部位框用于：

- 保存当前子部位的 box；
- 提供子部位自动标注或复核参考；
- 与 polygon 标签共同作为训练数据的一部分。

### 收缩松框

收缩松框是 Blink 自动收缩所需的起始框。

它不是普通父级框，也不应覆盖正式子部位标注框。它的用途是：

```text
用户画一个覆盖目标子部位的较宽松框
-> 程序用当前黄金 polygon 和松框生成 shrink trajectory
-> trajectory 用于训练子部位专家
```

因此主界面需要在画布上方工具栏提供明确的框工具：

```text
正式标注框
收缩松框
```

## 固定长宽比父级框

当前 Blink 流程会把父部位 ROI 统一映射到固定训练尺寸，例如 `512 x 512`。这种统一尺寸有利于训练，但对高分辨率分类学照片中的小部位细节可能产生损失。

新的设计改为：

```text
父级框在绘制时默认锁定长宽比
-> 不强行要求用户画任意形状框后再压缩变形
-> 子部位标注和映射尽量保留原始高分辨率细节
```

建议默认父部位比例模板：

```text
Head: 1:1
Mesosoma: 4:3
Gaster: 4:3 或 3:2
Whole body: 16:9 或自定义
```

实际比例应允许在设置中调整。

交互规则：

- 当前选中父部位时，框选工具默认锁定该父部位比例；
- 当前选中子部位时，框选工具默认自由比例；
- 当前处于“收缩松框”模式时，默认自由比例，后续可按子部位模板扩展；
- 应允许临时解除比例锁定，以适配特殊视角或姿态。

## 主工作台界面调整

### 工具栏

主标注工具栏保留现有工具，并新增或明确一个框工具：

```text
多边形
SAM 点选
SAM 框选
标注框
```

标注框的实际行为由当前部位角色和画布上方选中的框工具决定。

### 右侧部位树

右侧部位树承担父子关系的可见上下文。

用户选中部位时，界面需要显示：

```text
当前部位：Mandible
角色：子部位
父级上下文：Head
父级框：已存在 / 未框选
```

如果父级不明确，需要提示用户指定。

如果父级框不存在，需要提示用户先选中父部位并绘制父级框。

### 父子精修 / Blink 小区

在右侧 AI 区下方新增一个独立小区，避免和现有全图自动标注按钮混淆。

建议布局：

```text
父子精修 / Blink

当前路由：Head -> Mandible
父级框：已存在
路由专家：未指定        [配置路由专家]

画布上方框工具：
[SAM 提示框] [正式标注框] [收缩松框]

[自动标注子部位]
[执行自动收缩]
[训练当前子部位专家]
```

这个区域只放当前标注需要的高频操作。专家列表、候选专家、启用禁用和替换专家仍然放在设置中的路由专家配置里。

### 配置路由专家按钮

主界面只显示路由状态是不够的。用户看到“路由专家未指定”时，必须能直接进入配置。

按钮名称：

```text
配置路由专家
```

行为：

- 打开设置中的父子路由 / 路由专家配置；
- 自动定位到当前路由，例如 `Head -> Mandible`；
- 如果当前路由不存在，可以创建候选路由或提示创建；
- 当前未选子部位或父级不明确时置灰，并提示先选择子部位和父级上下文。

## 数据与状态规则

### 框角色

内部必须区分框角色：

```text
parent_context_box
child_annotation_box
shrink_loose_box
sam_prompt_box
```

界面可以使用更短的中文，但数据层不能混淆。否则后续训练数据会不可信。

### 全局坐标

合并后用户始终在主标注工作台工作。

因此正式标签应始终以全局图像坐标保存，不再需要用户点击 `Apply to Global`。

如果内部仍使用局部 ROI 做计算，映射过程应自动完成。

### 父子关系记忆

当用户为子部位确认父级上下文后，项目应继续记忆：

```text
child_part -> parent_part
```

这可以沿用当前 `blink_context_roi_parents` 的概念。

### 路由注册

当父子关系有效时，应继续注册或更新 cascade route candidate：

```text
parent_part -> child_part
```

路由专家是否启用和指定仍由路由专家配置面板管理。

### 自动收缩轨迹

执行自动收缩需要：

- 当前图像；
- 当前子部位；
- 当前父级上下文；
- 当前父级框；
- 当前子部位黄金 polygon；
- 当前子部位收缩松框。

轨迹保存时应保留 parent context：

```text
parent_part
parent_box
source
```

## 功能去留

保留：

- 父子路由关系；
- 父级 ROI 到子部位的上下文绑定；
- 子部位自动标注；
- 自动收缩轨迹生成；
- 当前子部位专家训练；
- 路由专家配置和候选专家管理；
- Blink 后端训练和推理能力。

隐藏或删除日常入口：

- 独立 Blink Workbench tab；
- `Open in Blink Workbench` 按钮；
- Blink 工作台内部重复部位列表；
- Blink 工作台内部重复日志；
- Blink switch 空格快捷键；
- 手动 `Apply to Global`。

保留为内部服务或临时兼容：

- 坐标映射逻辑；
- trajectory 生成逻辑；
- BlinkTrainingThread；
- cascade route 自动注册逻辑；
- 旧 Blink widget 可在过渡期保留，但不作为日常入口。

## 风险与保护

### 风险 1：框角色混淆

如果父级框、子部位框和收缩松框混在同一个字段里，训练数据会变得不可信。

保护：

- UI 在画布上方明确显示当前框工具；
- 数据层记录框角色；
- 自动收缩只读取 shrink loose box；
- 正式训练只读取正式标签和明确的 trajectory。

### 风险 2：父子关系不明确

用户选中子部位时，如果不知道父级是谁，自动标注和训练都会变得不稳定。

保护：

- 右侧状态块显示当前路由；
- 无父级时禁用 Blink 小区按钮；
- 提供“配置路由专家”入口和父级提示。

### 风险 3：固定比例框过度约束

特殊视角或姿态可能不适合默认比例。

保护：

- 默认锁比例，但允许临时解除；
- 比例模板可配置；
- 对子部位标注不继承父级比例。

### 风险 4：独立 Blink 功能一次性删除导致回退困难

保护：

- 第一阶段只隐藏日常入口，不删除核心类；
- 保留 `pre-blink-workbench-merge` git tag；
- 保留旧 Blink 测试到新主工作台功能稳定后再调整。

## 推荐实施策略

采用渐进式合并：

1. 保留 Blink 后端逻辑和旧 widget；
2. 在主工作台新增父子状态块，并把框工具集中放到画布上方；
3. 将父级固定比例框接入现有框选工具；
4. 将子部位自动标注、自动收缩、训练按钮接入主工作台；
5. 验证主工作台流程稳定后隐藏独立 Blink tab；
6. 最后再清理旧 UI 重复代码。

## 验收标准

合并后的研究者工作流应满足：

- 不需要进入独立 Blink Workbench 也能完成小部位精修；
- 选中子部位后能看见当前父级上下文；
- 父部位框默认按固定比例绘制；
- 子部位框不被父级固定比例限制；
- 收缩松框有明确模式，不污染正式标注框；
- 自动标注子部位能使用当前父子路由专家；
- 自动收缩能保存带 parent context 的 trajectory；
- 训练当前子部位专家后，可在设置中的路由专家配置中看到；
- 主界面路由状态旁有“配置路由专家”入口；
- 已有 2D/STL 标注、训练和导出流程不被破坏。
