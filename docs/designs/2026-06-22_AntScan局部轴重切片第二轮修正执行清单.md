## 文档定位

本文档是 Local Axis Reslice / 局部轴重切片进入第一轮代码实现后的第二轮修正清单。

2026-06-20 的设计稿和执行清单仍然保留，里面关于需求背景、数据规模、AI 必须进入第一版产品骨架、TaxaMask 数据模型归属、不能硬编码脑部、不能依赖体信号作为主定位算法等结论仍然有效。

但第一轮实现和试用反馈暴露出几个关键偏差：

- 三维体预览拖动明显卡顿，即使在 RTX 3090 上也不足够流畅。
- Local Axis Reslice 的主交互被推到了独立切片/MPR 页面里，和最初预想的“在 part volume 的三维体预览中显示和确认轴”有出入。
- MPR 点选区域过窄，界面横向被撑开，用户需要拖底部滚动条才能查看。
- 部位编号和显示名确认曾经出现连续弹窗，说明数据结构确认过程侵入了研究操作流程。
- 当前三维预览只能显示已导出 reslice 的 output Z axis，不能显示和编辑当前草稿 axis。

因此，后续不应继续在第一版 UI 上堆功能，而应先把工作流重心校正为：

```text
性能可用的 part volume 3D 预览
  -> source Z axis 锁定显示
  -> editable output Z axis 草稿显示与确认
  -> MPR/切片点选作为辅助精确输入
  -> local frame 生成
  -> 低分辨率临时 preview
  -> 确定性导出
  -> 操作记录进入训练素材
```

本清单在与旧清单冲突时，优先指导下一轮代码修改。

## 修正后的产品原则

- Local Axis Reslice 是 TaxaMask TIF part volume 工作流的一部分，不是搬运 `pca_head_orientation` 独立工作台。
- 功能名称继续使用 `Local Axis Reslice / 局部轴重切片`，不使用 `Brain Reslice`。
- 脑部只是第一个模板，不能把字段、UI 或算法写死为脑部专用。
- source Z axis 是原始 TIFF / part volume 的切片方向参考，只读、锁定、可显示，不可编辑。
- editable output Z axis 表示最终输出切片序列的推进轴，也就是 output Z axis，不等同于脑部解剖前后轴。
- 完整 local frame 必须保存 `origin, x_axis, y_axis, z_axis, output_axis, roll_reference`。
- roll reference point pair 是通用字段；脑部模板默认角色可以是 `left_eye/right_eye`，并且左右以标本自身解剖左右为准。
- 灰度 image 重切片使用 linear interpolation；mask/label 重切片使用 nearest-neighbor。
- 第一版默认 output spacing 继承 part spacing；isotropic spacing 放在高级选项，不作为默认。
- preview 第一版只做临时显示；正式保存的是导出的 `image.tif`、可选 `mask.tif` 和 `metadata.json`。
- 原始 TIFF、working volume、part image、part mask 都不被重切片流程覆盖。
- 体信号、折线图、QC 图不作为主定位流程；最多作为折叠的辅助导航、异常提示和样本质量检查。

## 当前问题复盘

### 1. 三维预览卡顿

当前体预览在鼠标拖动时仍可能触发较重的同步链路：

- 每次 mouse move 都进入 workbench 级 `render_volume_preview()`。
- 拖动中仍可能检查或生成 preview cache。
- mask boundary / masked image 模式下可能涉及 mask preview。
- axis overlay 跟随相机投影更新。
- GPU 渲染状态、纹理输入、UI 状态标签可能频繁同步。

这意味着性能瓶颈不只是显卡算力，而是交互链路过重。对于 2193 个 AntScan 个体的大规模处理场景，拖动时必须有轻量路径。

### 2. 主交互入口偏移

第一轮实现为了快速跑通闭环，新增了 Local Axis Reslice 页面和 MPR 点选界面。这对精确点位输入有帮助，但不应成为用户理解轴方向的主界面。

用户真正需要的是：

- 在当前 part volume 的三维形态上看到原始 source Z axis。
- 从 source Z axis 复制 editable output Z axis。
- 在三维体预览中看到当前草稿 axis。
- 在同一上下文里确认中心点和 roll reference。
- 必要时才打开大图 MPR 点选来补精确坐标。

### 3. 当前草稿和已导出结果没有分清

当前三维 overlay 更适合显示已存在记录，例如已导出的 reslice item。下一轮必须补上“draft local axis state”：

- 未导出前也能在三维预览看到 editable output Z axis。
- 用户调整 axis、origin、roll reference 后，三维 overlay 立即反映草稿状态。
- 导出后，草稿状态写成正式 reslice record。
- 左侧树只显示正式 reslice item；草稿详情放在右侧面板或当前页面状态中。

### 4. 数据记录要服务 AI 训练

AI 自动化仍然是第一版产品骨架的一部分，但当前阶段不强行选择模型。先要把人工操作变成标准训练素材：

- 用户如何从 full volume 创建 part。
- part 的 parent bbox / mask / contours。
- source Z axis。
- editable output Z axis 的初始值和最终值。
- origin。
- roll reference point pair。
- local frame。
- preview / export 参数。
- human_confirmed / usable_for_training。
- 操作时间、模板、软件版本、数据来源。

这样后续接入全局 ROI proposal 模型和 local frame proposal 模型时，TaxaMask 已经有可训练、可复核的数据结构。

## 阶段 B：3D 体预览性能修复

目标：让 part volume 3D 预览成为可长期使用的定位入口。3090 上应明显流畅，普通设备上也应可用。

重点文件：

```text
AntSleap/ui/tif_workbench.py
AntSleap/ui/tif_gpu_volume_canvas.py
tests/test_tif_workbench.py
```

核心策略：

- 拖动开始时进入 lightweight interaction mode。
- 拖动中只更新相机参数，不做重型 preview 重建。
- 拖动中降低 sample steps、texture target、supersample、detail enhancement。
- 拖动中默认暂停 mask boundary / masked image 的重计算，必要时临时退回 image-only 或 cached mask。
- 拖动中 overlay 只更新 2D 投影，不触发 volume texture 重传。
- 鼠标松开后启动 debounce timer，例如 150-300 ms 后再切回 still high quality。
- still high quality 重建时使用缓存，避免重复上传相同 texture。
- 状态栏明确显示 `拖动预览` / `静止高清`，便于用户理解为何拖动时画质较低。

建议新增或调整：

```text
_start_volume_interaction()
_finish_volume_interaction_debounced()
_render_volume_preview_lightweight()
_sync_gpu_camera_state_only()
volume_interaction_timer
volume_drag_preview_settings
```

验收方式：

- [ ] 鼠标拖动 3D volume 时不持续卡顿。
- [ ] 松开鼠标后自动恢复高清 still preview。
- [ ] 勾选/取消 mask boundary 或 masked image 不让拖动过程明显失控。
- [ ] 开启 `显示局部轴` 时拖动不卡到不可用。
- [ ] CPU fallback 下也不因频繁重算造成界面假死。
- [ ] 状态栏能显示当前是拖动预览还是静止高清。

风险：

- 如果 GPU offscreen renderer 每次 `set_render_state` 都触发完整 render，需要进一步拆成 camera-only update。
- 如果贴图上传仍频繁发生，需要在 `set_volume_data` 层增加数组 identity / shape / dtype / mode 缓存判断。
- 如果 mask boundary 生成本身太重，应只在 still 模式生成。

## 下一步推荐顺序

下一轮代码修改应按以下顺序推进：

1. 修复 3D 体预览拖动性能。
2. 建立 local axis draft 状态，并在 3D 预览中显示 draft output Z axis。
3. 把 Local Axis 主操作收回 part volume 右侧上下文。
4. 把 MPR 点选整理为辅助弹窗/全屏工具。
5. 补齐训练素材记录。
6. 再继续 AI proposal 和批量复核链路。

这能保证后续每一步都服务真实研究操作，而不是继续扩大一个不好用的临时界面。
