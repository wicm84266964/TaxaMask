# TaxaMask 2.4.0 TIF 工作台架构整理候选说明

日期：2026-07-08

建议版本号：`v2.4.0`

状态：本地候选版本说明，尚未发布到 GitHub Releases。

## 版本定位

本版本把 TIF/CT 工作台从“功能快速堆叠可用”整理为更适合长期维护的架构形态。重点不是改变研究者的日常入口，而是降低后续继续增加 TIF 导入、ROI、mask、训练、预测、3D 预览和 Local Axis 功能时的连锁故障风险。

对研究者可见的原则保持不变：

- 原始 TIF/CT stack 不被覆盖。
- AI 预测结果默认是待核验草稿，不会自动成为 `manual_truth`。
- 训练数据只使用人工核验后的训练真值。
- 后台任务完成前，保存、训练、预测、导入、ROI 创建和预览重建不会互相踩状态。

## 主要变化

- 拆分 TIF 工作台中的翻译表、弹窗、画布、helper 和 worker 模块，降低主文件职责密度。
- 新增 core 层数据安全 guard：
  - label role 写入矩阵。
  - truth promotion policy。
  - prediction import policy。
  - write intent guard。
- 新增 service/controller 层：
  - label save request。
  - truth promotion。
  - ROI part 创建请求。
  - backend 样本选择。
  - volume/mask preview request。
  - Local Axis draft/frame/reslice payload。
- 新增统一 task/state 层：
  - task context/state/manager。
  - Qt task adapter。
  - selection/edit/preview/backend/ROI/local-axis 状态摘要。
- 将高风险后台链路纳入统一 task manager：
  - label auto/manual save。
  - truth promotion。
  - ROI 创建。
  - backend train/predict。
  - TIF/Amira import。
  - materialize。
  - Local Axis export。
  - volume preview 和 mask preview。
- 增加 TIF 架构回归测试分组：
  - core safety。
  - service/task。
  - preview/export。
  - GUI key path。
  - research smoke。

## 研究流程收益

- 旧后台任务回调不会轻易刷新用户已经切换后的 specimen / part / reslice 界面。
- 自动保存、手动保存、训练、预测、导入和预览构建共享统一 busy lock，减少按钮状态和真实后台状态不一致。
- 预测导入、raw backup、manual truth promotion 和训练样本选择有集中 guard 和测试。
- agent context 可读取 `tif_task_summary` 和 `tif_state_summary`，更容易判断当前卡在保存、训练、预览还是导入。
- 后续新增 WebUI、CLI、批处理或新训练后端时，可以优先复用 core/service/task 层，而不是继续从 PySide Widget 里抽逻辑。

## 合并提醒

- 本版本不包含私有 CT/TIF 数据、项目数据库、模型权重或运行输出。
- 合并前仍应检查 `git status --short`，确认没有本地数据库、项目 JSON、生成 sidecars、run outputs 或 API 配置进入提交。
