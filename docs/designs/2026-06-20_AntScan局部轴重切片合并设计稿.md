# AntScan 局部轴重切片合并设计稿

日期：2026-06-20

## 设计定位

本文档用于记录 `pca_head_orientation` 原型项目合并进 TaxaMask 前的设计结论。

本轮目标不是把独立 PySide6 工作台搬进 TaxaMask，而是在 TaxaMask 当前 TIF 架构上确定一个面向 2193 个 AntScan 个体处理规模的成熟产品方案。

设计重心不能停留在“手工闭环先跑通，AI 留到未来”。在确认体信号路线无法可靠完成脑部定位后，AI 辅助自动化必须进入第一版产品骨架：

- 手工定轴仍是可靠标签来源和低置信样本复核入口。
- AI 负责提出全局目标部位候选和局部坐标系建议。
- TaxaMask 负责保存训练素材、管理模型版本、批量推理、人工复核和确定性重切片导出。

第一版不要求模型一次达到完全无人值守处理全部 2193 个体，但必须把训练、推理、复核、导出的链路作为正式功能设计，而不是作为遥远的后续扩展。

AI 接入必须纳入 TaxaMask 已有模型后端规范，而不是为 Local Axis Reslice 单独发明一套调用方式。也就是说，Local Axis 的训练、推理、模型 manifest、run 目录、日志、结果导入和人工复核状态，应与 2D/STL 父部位后端、Blink 子部位后端、现有 TIF volume backend 保持同一类规则：

- TaxaMask 写出 `contract.json`。
- 外部或内置后端读取 contract 执行 `prepare_dataset` / `train` / `predict`。
- 后端写回标准 result JSON 和 model manifest。
- TaxaMask 负责导入结果为 reviewable proposal，不允许后端直接改项目主 JSON。
- AI 输出永远先进入 proposal / draft 状态，经过 accepted / human confirmed 后才能驱动正式导出。

Local Axis 的特殊性只体现在任务输出不是 2D polygon 或 TIF label volume，而是 `global_roi_proposal` 和 `local_frame_proposal`。因此应做成 TaxaMask 统一模型规范下的 TIF/3D 任务扩展，而不是独立的“脑部 AI 插件”。

最终功能名称采用：

```text
Local Axis Reslice / 局部轴重切片
```

不采用 `Brain Reslice` 作为模块名。脑部只是第一个应用模板，底层能力必须保持通用，未来可以支持腿部腺体、胸部、腹部等其他局部结构。

## 背景与前提

`pca_head_orientation` 从创建开始就是为了未来合并进 TaxaMask 的测试分支。它最初针对 AntScan CT 数据中的脑部定位问题：

- AntScan 样本的原始 TIFF 起始页不一致。
- 原始切片方向不稳定。
- 个体在扫描容器中的姿态差异很大。
- 简单按原始 Z 轴做头尾判断无法稳定定位脑部。

原型项目早期尝试过基于体信号、端点窗口、大颚候选、眼候选、尾针候选来判断头尾方向，并生成 QC 图或 `head_up.tif`。这些探索对理解数据很有价值，但经过更多样本观察后，当前结论已经发生变化。

用户新检查的 18 个新下载个体显示：

- 小个体通常放在试管中，切片方向有时接近沿体轴，从头到尾或从尾到头。
- 较大个体从三维体预览看不像放在试管中，更像放在方形小盒中。
- 大个体的体态和朝向差异明显。
- 一些大个体不是沿体轴切片，而是横向切片；头、胸、腹会在同一组切片中同时被侧切。

因此，原始 TIFF Z 轴不能稳定视为蚂蚁体轴，也不能稳定视为头尾方向。体信号只能反映原始 Z 轴上的密度和结构变化，不能作为脑部或其他局部结构的主定位算法。

基于这个前提，需求从“脑部专用定位算法”升级为：

```text
在 TaxaMask 中实现通用的局部部位坐标系定义与重切片工作流。
```

## 现有 TaxaMask 能力判断

TaxaMask 当前已经具备部位提取的关键基础设施。合并时应复用这些能力，而不是在 `pca_head_orientation` 中重复实现。

当前可复用能力包括：

- 在 TIF workbench 中打开 AntScan TIFF。
- 显示 full volume。
- 在 Z/Y/X 多方向切片中浏览。
- 3D GPU 体预览。
- ROI 框选。
- 创建 part volume。
- 在 part volume 中画关键切片轮廓。
- 自动填充中间帧生成 part mask。
- 接受 part mask。
- 显示 3D mask boundary / masked image。
- 保存 part image、part mask、contours、extraction metadata。

关键文件：

```text
AntSleap/ui/tif_workbench.py
AntSleap/core/tif_part_extraction.py
AntSleap/core/tif_project.py
AntSleap/ui/tif_gpu_volume_canvas.py
```

其中 `AntSleap/core/tif_part_extraction.py` 已经提供：

```text
crop_volume_to_part(...)
add_rectangular_keyframe(...)
add_polygon_keyframe(...)
interpolate_masks_from_keyframes(...)
build_preview_mask_from_contours(...)
write_part_mask(...)
```

因此，Local Axis Reslice 不需要重新解决“如何从整只 TIFF 切出一个部位”的问题。它应建立在已有 part volume 之上。

## 需要合并的真正能力

从 `pca_head_orientation` 合并进 TaxaMask 的核心不是旧的预测流程，而是以下概念和能力：

```text
source Z axis reference
-> editable output Z axis
-> origin / reslice center
-> roll reference point pair
-> complete local frame
-> part-local reslice preview
-> part-local resliced TIFF export
```

其中：

- source Z axis 是原始 TIFF 切片方向参考，锁定、只读、不可编辑。
- editable output Z axis 是用户从 source Z axis 复制出来后编辑的局部输出切片推进轴。
- origin 是旋转中心和重切片中心。
- roll reference point pair 用于决定绕 output Z axis 的旋转方向。
- local frame 必须完整保存，不能只保存一根轴。
- 重切片只作用于当前 part volume，不默认重切整只 TIFF。

脑部模板中，roll reference point pair 可以默认使用：

```text
left_eye / right_eye
```

但底层字段应保持通用：

```text
roll_reference_point_pair
```

left/right 以标本自身解剖左右为准，不以屏幕视角为准。

## 为什么不把功能直接塞进现有 TIF 主工作台

当前 TIF workbench 已经承担很多任务：

- 导入 TIF。
- full volume 查看。
- part ROI。
- part mask。
- 3D 预览。
- 标注层。
- 材料表。
- 训练导出。
- 外部预测导入。

Local Axis Reslice 是一个专司定位、定轴、重切片的复杂工作流。如果直接把它作为几个按钮堆进当前右侧栏，会让主工作台变得过重，也会混淆研究流程。

推荐界面策略：

```text
TIF Workbench：
  负责导入 TIFF、创建 specimen、创建 part、生成/接受 part mask。

Local Axis Reslice Page：
  负责在已选 part volume 内定义局部坐标系，并导出重切片结果。
```

也就是说，Local Axis Reslice 应是 TaxaMask 内的专用页面或专用工作台模式，而不是独立项目，也不是 TIF 主工作台右侧栏里的一个杂项面板。

## 推荐总体工作流

第一版目标不是只让用户在一个 part 上手工导出一次 TIFF，而是形成一个可处理上千个体的闭环：

```text
人工可靠标定
  -> 结构化训练素材
  -> AI 全局 ROI / 局部坐标系建议
  -> 批量预览与置信度分流
  -> 人工复核低置信样本
  -> 确定性重切片导出
  -> 修正结果回流训练集
```

其中 Local Axis Reslice 页面要同时支持两种使用方式：

- 单样本精修：用户手工创建或修正 part、axis、roll reference，导出可信结果。
- 批量复核：AI 对多个 specimen 提出候选，用户按队列接受、微调或拒绝。

单样本基础工作流：

```text
主 TIFF specimen
  -> 使用 TaxaMask 现有 ROI / 关键切片 / mask 流程创建目标 part
  -> 打开 Local Axis Reslice 专用页面
  -> 在 part volume 中显示锁定 source Z axis
  -> 从 source Z axis 复制 editable output Z axis
  -> 设置 origin / reslice center
  -> 设置 roll reference point pair
  -> 计算完整 local frame
  -> 生成临时低分辨率 reslice preview
  -> 导出 image.tif + metadata.json
  -> 如 part 已有 mask，可选导出 mask.tif
  -> 输出挂在 specimen -> part -> reslices 下
```

AI 辅助批量工作流：

```text
1. 在 TaxaMask 中导入或打开一批 AntScan TIFF。
2. 对一批代表性样本进行人工可靠标定，保存训练素材。
3. 训练或加载全局 ROI proposal 模型和局部 local frame 模型。
4. 对未处理 specimen 批量运行 AI proposal。
5. 左侧或专用队列显示每个样本的状态：
   no_proposal / proposed / needs_review / accepted / exported / rejected。
6. 用户优先复核低置信、高风险或异常样本。
7. 高置信样本可以批量生成 preview，人工抽查后导出。
8. 人工修正结果写回 metadata，并可加入下一轮训练集。
9. 所有正式输出仍挂在 specimen -> part -> reslices 下。
```

第一版必须覆盖的成熟产品闭环：

```text
1. 单样本手工 Local Axis Reslice。
2. 每次确认自动保存可训练 metadata。
3. 训练集 manifest 导出和模型输入准备。
4. 模型 proposal 结果的项目内登记。
5. 批量 proposal 队列。
6. 低置信人工复核。
7. 接受 proposal 后确定性导出 image.tif + metadata.json。
8. mask.tif 有条件导出，不阻塞 image 闭环。
```

## 左侧树组织

用户已确认接受以下结构：

```text
Specimen
  Full volume
  Head part
    Reslices
      brain_axis_v1
      local_axis_v2
  Leg gland part
    Reslices
      local_axis_v1
```

设计原则：

- `Reslices` 是 part 下的派生输出集合。
- reslice item 不展开过深。
- 详细参数放到右侧 Local Axis Reslice 面板。
- 左侧树负责导航数据对象，右侧面板负责当前对象操作。

## Local Axis Reslice 页面信息架构

推荐专用页面分区：

```text
1. Current part
   当前 specimen、part、shape、spacing、parent bbox、mask 状态

2. Source reference
   source Z axis：锁定、只读、始终可见

3. Editable output axis
   从 source Z axis 复制
   设置 output Z axis
   设置 origin / reslice center
   支持数值微调和视图交互

4. Roll reference
   point pair
   模板默认标签
   脑部模板：left_eye / right_eye

5. Local frame
   显示计算后的 x_axis / y_axis / z_axis
   显示是否正交、是否单位化、是否有效

6. Preview
   生成临时低分辨率 preview
   不作为第一版正式保存输出

7. Export
   image.tif + metadata.json
   mask.tif 可选
   标记 human_confirmed / usable_for_training

8. Optional diagnostics
   Source Z Signal，默认折叠，后续再做
```

第一版可以先使用表单 + 2D/MPR 点选 + 简单 overlay。复杂 3D gizmo 可以后续再增强。

## 第一版成熟产品范围

第一版应被定义为“可开始规模化处理 AntScan 数据集的产品骨架”，而不是单样本实验工具。

必须包含：

```text
1. 单样本 Local Axis Reslice 页面
2. 批量 proposal / review 队列
3. 全局 ROI proposal record
4. local frame proposal record
5. human confirmed record
6. training manifest 导出
7. model registry / model version 记录
8. selected specimens / batch proposal 入口
9. accepted proposal 的确定性 reslice export
10. rejected / needs_review / hard_case 状态管理
```

可以接受第一版模型能力还不完美，但不能接受第一版没有模型接入位置、没有批量状态、没有人工修正回流、没有训练素材导出。

研究工作流上的含义：

- 用户不应在 2193 个体上逐个从零手工定轴。
- 第一批人工样本应立即转化为训练集。
- AI 先覆盖“建议”和“排队”，高置信结果减少重复劳动，低置信结果交给人工。
- 人工修正不是额外负担，而是下一轮模型训练数据。

第一版建议在 UI 中明确区分三类结果：

```text
Manual confirmed
AI proposed
AI proposed + human edited
```

正式导出的 reslice 必须来自 accepted / human confirmed 状态，不能从未复核 proposal 静默导出。

## 轴与坐标系定义

### Source Z Axis

source Z axis 表示原始 TIFF 切片方向：

```text
Z0 -> Z last
```

它的含义是“扫描文件中的原始页序方向”，不是头尾方向，也不是解剖方向。

规则：

- 始终可见。
- 锁定。
- 不可编辑。
- 不修改原始 TIFF。
- 可以作为创建 editable output axis 的初始参考。

### Editable Output Z Axis

editable axis 表示最终输出切片序列的推进轴，也就是 output Z axis。

它不是脑部解剖前后轴。脑部或其他部位的完整方向由 output Z axis 和 roll reference 共同决定。

第一版可以支持：

- 从 source Z axis 复制。
- 设置起点和终点。
- 设置方向向量。
- 设置 origin。
- 数值微调。
- 在切片/MPR 上点选位置。

### Origin

origin 是：

- 重切片中心。
- 旋转中心。
- 输出 volume 的中心或参考中心。

应保存为 part volume 坐标：

```text
origin_zyx
```

同时可以根据 parent bbox 推导 full volume 坐标。

### Roll Reference Point Pair

一根 output Z axis 只能决定切片序列推进方向，不能决定切片平面内的旋转方向。必须使用 roll reference 消除绕轴旋转的不确定性。

脑部模板默认：

```text
label_a: left_eye
label_b: right_eye
```

其他部位模板可以使用不同成对点，例如左右边界、腺体两端、关节两侧等。

### Local Frame

最终必须保存完整局部坐标系：

```text
origin
x_axis
y_axis
z_axis
output_axis
roll_reference
```

保存一根轴是不够的。否则以后无法复现统一方向的切片平面，也无法训练 AI 预测同样的目标。

## 数据保存结构

推荐输出路径：

```text
specimens/<specimen_id>/parts/<part_id>/reslices/<reslice_id>/
  image.tif
  metadata.json
  mask.tif        可选
```

不建议第一版保存 preview。preview 只用于用户确认，正式保存的是导出的重切片结果和 metadata。

项目 JSON 中，reslice 记录建议挂在 part metadata 下：

```text
specimen.parts[].metadata.local_axis_reslices[]
```

这样不用破坏现有 part record 结构，也能在未来增加 helper 方法统一管理。

同时第一版需要新增或约定以下项目级/part 级记录：

```text
specimen.metadata.local_axis_global_proposals[]
part.metadata.local_axis_frame_proposals[]
part.metadata.local_axis_reslices[]
project_data.models.local_axis_models[]
project_data.runs.local_axis_runs[]
```

建议含义：

- `local_axis_global_proposals`：模型 A 或人工创建的目标部位 bbox / center 候选。
- `local_axis_frame_proposals`：模型 B 在 part 内提出的 origin / axis / roll reference 候选。
- `local_axis_reslices`：已接受并导出的正式重切片记录。
- `local_axis_models`：已登记的 proposal 模型、版本、模板、输入输出约定。
- `local_axis_runs`：一次批量 proposal / training export / reslice export 的运行记录。

这几个记录可以先存在 metadata 和 runs/models 的扩展字段中，不必立刻改变项目 schema version，但需要统一 helper 函数读写，避免 UI 任意拼接 JSON。

## 训练素材记录设计

用户明确希望从手工定轴中积累可训练素材。这个设计不是附属记录，而是第一版 AI 自动化链路的基础。

每次用户确认一次 Local Axis Reslice，应自动形成一条训练样本，并能进入 training manifest。该样本包含两类标签：

```text
1. 全局定位标签
2. 局部定轴标签
```

### 全局定位标签

用于训练或评估“从整只 TIFF 找 head / brain / target part”的模型：

```text
specimen_id
part_id
parent_bbox_zyx
part center in full volume
part image path
source working volume path
spacing_zyx
hard_case_flags
```

其中 `parent_bbox_zyx` 很重要。它表示人工确认的目标 part 在整只 TIFF 中的位置，是全局 ROI proposal 模型的基础。

### 局部定轴标签

用于训练或评估“局部坐标系预测器”：

```text
origin_zyx
output_axis_start_zyx
output_axis_end_zyx
roll_reference point pair
local_frame x/y/z axes
output shape
output spacing
```

这些标签来自人工确认后的最终结果，而不是临时拖动过程。

### 质量标签

不是所有样本都适合训练。应允许用户或后续审核标记：

```text
usable_for_training: true / false
review_status: accepted / needs_review / rejected
hard_case_flags:
  - tube_sample
  - box_sample
  - transverse_slice
  - poor_contrast
  - damaged_sample
  - support_interference
  - uncertain_eye_reference
```

这些字段对 2193 个 AntScan 个体尤其重要。第一版训练集导出就应支持只用高质量样本训练、用困难样本做验证或失败分析。

### 操作过程日志

操作过程日志不是第一训练标签，但有助于追溯和改进 UI。

第一版可记录简化事件：

```text
created_from_source_z_axis
axis_adjusted
origin_set
roll_reference_set
preview_generated
export_confirmed
```

无需记录每一次鼠标移动。重点是最终确认结果和关键操作节点。

## 第一版 AI 自动化产品路线

用户实际目标是处理 AntScan 上千个不同体态、不同朝向的个体，并准确找出脑部后按统一方向切片。

这个任务不应期待一个单一模型从整只 TIFF 直接输出最终切片。更合理的是在第一版就建立分阶段 AI 产品链路：

```text
Full volume
  -> 全局 head / brain / target part proposal
  -> part volume
  -> local frame predictor
  -> deterministic reslice
  -> high-confidence auto export / low-confidence human review
```

第一版建议包含两个 AI 组件位点，即使早期模型可以先是 baseline 或外部脚本接入：

```text
模型 A：全局 ROI / head-brain proposal
输入：整只 TIFF 的低分辨率 volume
输出：目标部位 bbox 或 center

模型 B：局部 local frame predictor
输入：裁剪后的 part volume
输出：origin、output Z axis、roll reference points
```

### 模型 A：全局 ROI / 目标部位候选

模型 A 的任务是从 full volume 中提出目标部位候选，例如 head / brain-near ROI。

第一版可接受的实现层级：

```text
Level 0：人工 ROI + 保存训练标签
Level 1：基于已有 part records 训练/运行外部 proposal 脚本
Level 2：TaxaMask 内部封装模型推理入口
```

无论模型是否成熟，TaxaMask 第一版都应设计好这些数据对象：

```text
global_proposal_id
specimen_id
target_template_id
bbox_zyx
center_zyx
confidence
model_id
model_version
status: proposed / accepted / rejected / needs_review
hard_case_flags
```

模型 A 的输出不直接覆盖 part。它先作为 proposal 存在。用户接受后，TaxaMask 才用现有 `crop_volume_to_part(...)` 创建或更新 part volume。

### 模型 B：局部坐标系预测器

模型 B 的任务是在已经裁剪出的 part volume 中预测局部定轴所需的点和轴。

第一版推荐模型形态：

```text
3D U-Net heatmap landmark detector
```

输入：

```text
part image volume
part mask，可选
spacing_zyx
template_id，例如 brain
```

输出：

```text
origin heatmap
output_axis point / endpoint heatmap
left_eye heatmap
right_eye heatmap
可选其他模板点
```

TaxaMask 将模型输出转换为：

```text
origin
editable output Z axis
roll_reference point pair
local_frame
confidence
proposal_status
```

模型 B 的输出也不直接成为最终结果。它先成为 local frame proposal，用户可以接受、微调或拒绝。

### 第一版 AI 交互要求

Local Axis Reslice 页面需要同时显示：

```text
AI proposal
human confirmed result
diff / deviation
confidence
quality flags
```

用户操作应包括：

```text
Accept proposal
Edit proposal
Reject proposal
Mark usable for training
Mark hard case
Export after accept
```

对 2193 个体来说，必须有批量队列：

```text
All
No proposal
Proposed
Needs review
Accepted
Exported
Rejected
Hard cases
```

这样 AI 自动化不是隐藏在后台的算法，而是研究者能复核、能修正、能审计的工作流。

### 第一版训练功能要求

第一版至少需要提供：

```text
Export Local Axis Training Set
Import / Register Local Axis Model
Run Proposal on Selected Specimens
Run Proposal on Batch
Review Proposal Queue
```

训练本身可以先通过外部脚本或后端命令运行，但 TaxaMask 必须掌握训练数据 manifest、模型登记、推理结果登记和人工复核状态。

这意味着第一版不是“AI 模型已经完美”，而是“AI 自动化链路已经作为产品骨架落地”。模型可以迭代，但数据结构和交互不能等到以后再补。

## 统一模型接入规范

Local Axis AI 应复用 TaxaMask 已经形成的模型接入思想，而不是成为一个旁路系统。

现有 TaxaMask 中已经有三类可参考规则：

```text
2D/STL external parent backend
  docs/contracts/external_backend_contract_v1.md

Blink child-part backend
  docs/contracts/external_blink_backend_contract_v1.md

TIF volume backend
  AntSleap/core/tif_backend.py
```

这些规则的共同点是：

- TaxaMask 生成 contract。
- 后端脚本通过 `{contract}` 或 `{contract_json}` 接收任务。
- 任务有清晰 action。
- 运行目录里保留 contract、logs、outputs、result。
- 训练后写 model manifest。
- 推理结果进入 TaxaMask 作为可复核草稿，而不是覆盖人工真值。

Local Axis 应沿用这些共同点。建议新增一个任务专用契约，而不是直接复用现有 TIF label volume 契约：

```text
docs/contracts/tif_local_axis_backend_contract_v1.md
```

原因是现有 `AntSleap/core/tif_backend.py` 的输出重点是 `prediction_label_volume`，用于 TIF 分割草稿；Local Axis 的输出是目标部位候选和局部坐标系候选，语义不同。如果强行塞进 label volume contract，会让后端和 UI 都需要猜测 artifact 含义。

推荐命名：

```text
contract schema: taxamask_tif_local_axis_backend_contract_v1
result schema:   taxamask_tif_local_axis_backend_result_v1
manifest schema: taxamask_tif_local_axis_model_manifest_v1
```

推荐 action：

```text
prepare_dataset
train
predict_global_roi
predict_local_frame
predict
```

其中 `predict` 可作为组合入口：后端可以同时返回 global ROI proposals 和 local frame proposals；TaxaMask 导入时按 artifact type 分流。

推荐 artifact type：

```text
local_axis_training_manifest
local_axis_model_manifest
global_roi_proposals
local_frame_proposals
```

关键规则：

- 后端不得直接修改 TaxaMask project JSON。
- 后端不得直接创建正式 part 或正式 reslice。
- `global_roi_proposals` 只能写入 specimen 级 proposal 列表。
- `local_frame_proposals` 只能写入 part 级 proposal 列表。
- 用户接受 global proposal 后，TaxaMask 再调用现有 part extraction 创建或更新 part。
- 用户接受 local frame proposal 后，TaxaMask 再执行确定性 reslice export。
- 训练数据来自 human confirmed / usable_for_training 的记录，AI proposal 未复核前不能进入训练真值。

对模型方案系统的影响：

Local Axis 模型应作为 TIF/3D workflow 的 model profile 子项保存，而不是散落在单个按钮配置里。模型记录至少应说明：

```text
profile_scope: tif_local_axis
backend_type: builtin_baseline | external_local_axis
template_id: head / brain / custom_part
global_roi_model_manifest
local_frame_model_manifest
train_command
predict_command
active_model_id
inference_params
```

这样未来用户在 TaxaMask 里会看到统一的逻辑：

```text
选择模型方案
-> 导出训练集 / 训练或登记模型
-> 对选中 specimen 或批量 specimen 运行 proposal
-> 人工复核 proposal
-> accepted 后导出正式结果
```

对研究工作流的意义是：不同模型可以更换，但项目中每一次“哪个模型给了什么建议、用户是否接受、最后导出了什么”都能追溯。对于 2193 个个体的数据集，这比单次模型准确率本身更重要，因为它决定了后续能否批量审计、返工和继续训练。

## 体信号与 QC 图取舍

### 体信号的重新定位

体信号不应作为主定位算法。它受体态、容器、切片方向影响太大。

可保留的价值：

- 原始 Z 轴导航。
- 快速跳页。
- 判断主体是否贯穿几乎整段 TIFF。
- 发现容器、支撑、胶质干扰。
- 发现 TIFF 页数异常或信号断裂。

如果后续迁移，应命名为：

```text
Source Z Signal / 原始 Z 信号
```

默认折叠，只作为辅助诊断，不参与 local frame 计算。

### QC 图

QC 图第一版不迁移。

原因：

- 信息量太重。
- 原先服务的是端点头尾判断算法。
- 当前数据观察已经削弱了“原始 Z 轴接近体轴”的假设。
- 对 Local Axis Reslice 主流程帮助有限。

QC 图和旧体信号预测代码可以保留作参考，不进入正式用户界面。

### 原型右侧栏取舍

不迁移：

```text
Orientation Processing
endpoint pages
full-axis z step
generate QC
generate head_up.tif
head side
needs flip
confidence
mandible/stinger candidate as direction decision
旧人工标注表单
```

可参考但降级：

```text
点击信号跳转原始 Z 页
原始 Z 轴可视化
异常样本 flags
样本质量检查经验
```

## 风险与待确认点

### UI 风险

- 现有 TIF workbench 已经复杂，不能继续堆按钮。
- 专用页面需要明确和主 TIF workbench 的边界。
- 左侧树增加 reslice item 后，不能让层级过深。

### 数据风险

- 如果只保存 output axis，不保存 full local frame，未来无法复现切片方向。
- 如果不保存 parent bbox，未来无法训练从 full volume 找脑部的模型。
- 如果不保存 quality 标记，训练数据会混入不可靠样本。

### 几何风险

- Z/Y/X voxel spacing 可能不一致，local frame 计算要考虑物理比例。
- output Z axis 和 roll reference 可能接近平行或退化，需要校验。
- 左右眼点位如果标反，会导致 roll 方向错误。

### 重采样风险

- 灰度 image 可以 linear interpolation。
- mask / label 必须 nearest-neighbor。
- 不能覆盖原始 TIFF、working volume 或 part image。

### 自动化风险

- 体信号不能承担主定位。
- AI 需要多样人工样本，不能只训练好看的试管小个体。
- 横向切片、大个体盒装样本必须进入训练和验证集合。

## 明确不做

第一阶段明确不做：

- 不把独立 PySide6 工作台整体搬进 TaxaMask。
- 不继续使用 `pca_head_orientation` 的 WorkbenchProject JSON 作为主数据模型。
- 不跨项目 import `taxamask_volume_adapter.py`。
- 不把模块命名为 Brain Reslice。
- 不把脑部逻辑写死。
- 不迁移 QC 图到正式界面。
- 不把体信号作为主定位算法。
- 不生成或依赖 `head_up.tif`。
- 不默认重切整只 TIFF。
- 不覆盖原始 TIFF。
- 不覆盖 part image。
- 不自动把重切片结果写入 manual truth。
- 不要求第一版模型达到完全无人值守处理全部样本。
- 不允许 AI proposal 未经 accepted / human confirmed 状态就静默成为正式导出。
- 不把 AI 作为黑箱直接输出最终 TIFF；AI 输出必须转为可复核 proposal。

## 总结

本次合并的正确边界是：

```text
复用 TaxaMask 已有 part extraction；
新增 part 内 source Z reference、editable output axis、roll reference、local frame metadata、AI proposal、批量复核队列和 part reslice 导出。
```

`pca_head_orientation` 的历史价值在于证明：

- AntScan 原始 Z 轴不能稳定代表体轴。
- 体信号不适合作为脑部主定位算法。
- 脑部需求应升级为通用局部轴重切片。
- 手工确认的局部坐标系必须从第一版开始结构化保存，直接进入 AI 训练和复核闭环。

因此，第一版不能只是一个小而完整的人工闭环。它应同时完成：

```text
人工可靠标定
训练素材保存
AI proposal 接入
批量复核队列
accepted 后确定性导出
```

模型效果可以迭代，但产品链路必须从第一版开始面向 2193 个体的规模化处理。
