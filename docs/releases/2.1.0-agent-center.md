# TaxaMask 2.1.0 Agent Center Update

Release date: 2026-07-04

Branch: `main`

Tag: `v2.1.0`

This release updates the embedded first-party Ant-Code Agent Center used by TaxaMask. It keeps the TaxaMask PDF, 2D/STL, and TIF/CT research data contracts unchanged, but improves the reliability of long Agent Center sessions, background terminal work, context recovery, and embedded runtime version reporting.

The embedded Ant-Code runtime in `vendor/ant-code/` is aligned to Ant-Code `1.2.4`.

## Research Workflow Impact

The main goal of this release is a smoother and more recoverable Agent Center experience during real research sessions.

For TaxaMask users, Agent Center is often used while screening PDFs, importing candidate images, checking profiles, diagnosing backend configuration, starting long-running extraction or training commands, and repairing the local source tree if the main GUI cannot start. Those tasks can run for a long time and may depend on large prompts, local terminal jobs, or interrupted model responses. This update makes those cases easier to monitor, cancel, and resume.

## Embedded Ant-Code Runtime

- Aligns the embedded runtime version with Ant-Code `1.2.4`.
- Keeps TaxaMask's embedded configuration boundary: the default workspace remains the TaxaMask repository, and this update does not adopt the standalone Ant-Code global model configuration path.
- Preserves the TaxaMask source-write guard and recovery Dashboard behavior.

## Long Tasks And Background Terminals

- Adds background terminal task listing for the current session/workspace.
- Adds background terminal cancellation through the same registered task cleanup path used by the Dashboard.
- Teaches Agent profiles to use registered background terminal tasks for long-running commands instead of ad hoc detached shell processes.
- Adds guidance for checking existing background terminal tasks before restarting servers, viewers, downloads, renders, or batch jobs.

For researchers, this means long PDF extraction, candidate import, preview generation, training, or local service commands can be started in the background and later reviewed or cancelled from the Agent Center path instead of relying on operating-system process hunting.

## Context And Recovery Reliability

- Raises gateway retry defaults for the embedded runtime.
- Adds total gateway request timeout and streaming idle timeout handling.
- Makes streaming response parsing and error preview handling more tolerant of gateway differences.
- Aligns context byte budgets with model token budgets.
- Allows full prompt compaction ratio when configured for large-context models.
- When resuming a compacted session, keeps the compacted summary if restoring the full archived transcript would exceed the prompt budget.
- Persists partial assistant draft text if a gateway request fails after text has already streamed.

For daily work, this reduces the chance that a large Agent conversation, failed gateway response, or resumed long session loses useful context at exactly the moment when the user is trying to diagnose a PDF batch, backend error, or source edit.

## Dashboard And Session Behavior

- Rebuilds context-window limits when model/context configuration changes while preserving existing summaries.
- Releases stuck running-turn state after Dashboard interruption if the normal cleanup path does not finish quickly.
- Adds background terminal snapshots to Dashboard background task state.
- Routes Dashboard recycle actions to the terminal cancellation API when the item is a terminal task.

## Validation

Validation performed for the embedded TaxaMask integration:

- `npm run check:syntax` inside `vendor/ant-code`
- `npm run verify:gateway` inside `vendor/ant-code`
- `npm test` inside `vendor/ant-code`
- Smoke checks for embedded version reporting, built-in tool duplication, background terminal listing, and Dashboard cancellation runtime availability
- Manual embedded smoke check confirming a background terminal task can be started and cancelled, and normal Agent replies still work

The embedded Ant-Code package currently has no formal Node test cases, so `npm test` reports 0 tests while still completing successfully.

## Data Boundary

This release does not include private research data.

It does not change TaxaMask project storage, PDF evidence databases, annotation truth data, TIF sidecars, model weights, API keys, or user runtime configuration.
