# TaxaMask 2.3.2 Taxonomy Paper Finder 统一文献工作流

日期：2026-07-12

版本号：`v2.3.2`

这是 `v2.3.1` 之后的小型功能更新。它将原先只负责开放 PDF 获取的内嵌 Skill 升级为完整的 `taxonomy-paper-finder`，把每日文献推荐、专题检索、证据筛选、精选下载和批量获取放进同一条可审计路线。

## 研究工作流影响

- 已有 PDF 的研究者可以直接继续使用 PDF Screener 和图文提取流程。
- 尚未收集 PDF 时，Agent 会先确认任务是每日推荐、专题检索、获取已筛选文献，还是批量采集。
- 每日推荐通常保留约 5 篇主要文献、最多 8 篇候选和 3 篇深读；结果不足时不会用弱文献凑数。
- 下载前必须由研究者确认。从筛选报告衔接下载时默认只处理 `deep-reads`，避免自动下载所有候选。
- 检索、筛选决定、digest、DOI、来源 URL 和下载 manifest 共同构成文献来源审计链。

## 统一后的能力

- arXiv、bioRxiv 和 PubMed 的近期候选发现。
- OpenAlex、Crossref 和 Europe PMC 的专题或历史文献检索。
- screening report 契约验证以及 JSON、Markdown、HTML digest 导出。
- 将主要推荐、候选或深读文献桥接为 `records.csv` 和 `doi_list.txt`。
- 合法开放 PDF 的 search-only、dry-run、有界下载和断点续跑。
- DOI 或规范化标题去重，以及逐条下载状态和失败原因记录。

## 数据与安全边界

- 只使用开放元数据和合法暴露的 PDF 链接，不使用 Sci-Hub、LibGen、付费墙绕过、验证码绕过或登录态抓取。
- 内嵌版默认把运行产物写入 `TaxaMask_outputs/taxonomy-paper-finder/`，不写入 `vendor` 源码目录。
- paper records、screening report、digest、manifest 和 PDF 都是本地研究材料，不自动进入 Git。
- PDF 文献、图版和描述仍属于 evidence/provenance，不会自动成为 2D/STL 训练真值或 TIF `manual_truth`。

## 验证

- 12 条相关单元测试通过，覆盖 Skill 完整性、精选文献桥接、默认深读范围、运行目录隔离、Agent 上下文和 PDF evidence provenance。
- 真实 TaxaMask Qt 环境中的 PDF Agent 上下文测试通过。
- Ant-Code Skill 注册器只发现 `taxonomy-paper-finder`，不再发现旧 `taxonomy-pdf-harvest`。
- Python 脚本语法和 JSON 配置/schema 检查通过。
- 真实 CLI 干跑把一篇深读文献转换为下载输入并写入 `dry_run` manifest，没有下载 PDF。
- 开发仓中的 Ant-Code 自检通过 118 个文件语法检查和 62 个 Dashboard 资源逐字节校验。

## 升级说明

从 `v2.3.1` 升级不需要迁移 TaxaMask 项目、数据库或模型。旧 Skill 名称已从当前 Agent 路由中移除；后续应使用 `taxonomy-paper-finder`。
