# TaxaMask 第五轮优化执行清单

## 0. 清单状态

- 日期：2026-07-18
- 状态：M0 至 M5 已完成；2026-07-21 最终需求对齐审计通过
- 当前分支：`codex/experimental-research-traceability`
- 设计依据：`docs/designs/2026-07-18_TaxaMask第五轮优化设计稿.md`
- 来源依据：专业人士《TaxaMask_项目分析报告.md》、仓库事实核验、用户最终确认
- 确认方式：用户确认关键研究规则，并授权其余工程建议按推荐方案整体执行；不虚构用户曾逐项确认暂缓项
- 明确排除：多标注者系统、平级 JSON/SQLite `StorageBackend`、主入口统一迁移、Mixin `Protocol`、整体重写 Manager
- 网格存储口径：项目 SQLite 是唯一导出台账和恢复依据；默认不生成网格 JSON sidecar；OBJ/PLY 不进入本轮

本清单是第五轮优化的执行路线。任务编号用于追踪，阅读时只需先看任务名称。任何任务只有在验证证据已保存后才能勾选完成。

## 1. 总路线

| 顺序 | 阶段 | 对研究工作的直接意义 | 完成后暂停检查 |
|---:|---|---|---|
| 1 | 锁定现状与安全边界 | 防止优化过程中悄悄改变已有结果或真值规则 | 是 |
| 2 | 文件完整性与完整训练记录 | 能回答模型用了哪些文件、参数和环境 | 是 |
| 3 | 推理流程拆分与诊断日志 | 能解释 Locator、专家路由或 SAM 为什么成功/失败 | 是 |
| 4 | Blender 5.0 / STL 网格导出 | 将审核标签体可靠地用于外部渲染、检查和潜在测量 | 是 |
| 5 | Local Axis 高风险样本优先复核 | 优先检查更可能出错的样本，同时保护预测结果批量通过和 Local Axis 单项接受 | 是 |
| 6 | 整体测试、文档和交付验收 | 确保跨平台、数据安全和研究流程没有回归 | 最终验收 |

## 2. 全程执行规则

- [x] 每次只推进当前阶段，不跨过阶段验收门。
- [x] 不修改现有人工真值规则：未验证、未通过的样本永不进入训练。
- [x] 不改变“一键通过”的用户主动操作语义，系统不得自动替用户通过。
- [x] 不把真实 CT、数据库、模型权重、运行日志、API 配置或用户项目提交到 Git。
- [x] 临时验证产物统一放入 `.tmp_validation/`，阶段结束前清理。
- [x] 新运行产物使用项目级或 run 级目录，不在仓库根目录散落文件。
- [x] 获准存在的非权威 JSON 产物必须有 `schema_version` 和向后兼容说明；网格导出默认不生成 JSON sidecar。
- [x] 写文件采用原子写入；获准存在的 JSON 报告优先复用 `AntSleap/core/safe_io.py`。任何普通 JSON 产物都不能恢复、覆盖或替代 SQLite 项目状态。
- [x] 推理拆分阶段不调整算法阈值、模型权重或输出语义。
- [x] 每个阶段结束后先汇报改动、产物位置、验证结果和剩余风险，再进入下一阶段。

- 完整的数据集发布平台、引用标识和线上共享系统；本轮仅交付可追溯 STL、SQLite 来源记录和文档基础。

## 10. 中止并请示条件

执行中遇到以下情况必须停止当前阶段并向用户说明：

- 需要改变人工真值或一键通过的现有语义；
- 需要批量改写、迁移或删除现有用户项目；
- 需要默认平滑、补洞或删除可能有研究意义的小结构；
- Blender 5.0 的比例或轴向无法从现有 metadata 可靠还原；
- 大型 CT 哈希或网格生成造成不可接受的时间、内存或磁盘压力；
- 推理拆分后的结果超出约定容差；
- 新依赖显著改变安装体积、平台支持或 GPU 要求；
- SQLite 导出记录与 STL 文件出现无法按既定状态机安全复核的分歧；
- 需要把未验证样本用于训练才能继续功能设计；
- 发现用户数据、密钥或模型权重可能进入 Git。
