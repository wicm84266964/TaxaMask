## 研究工作流中的变化

- 单张和批量预测结果保存方式不变，候选框与 polygon 数值对照一致。
- 运行日志仍在 `TaxaMask_outputs/runtime_logs/`，没有第二套日志系统。
- 现在可以从事件看出某部位是低峰跳过、未配置路由、专家不可用/低于门槛、SAM 无结果还是成功。
- 批量预测某张图失败时会显示具体图像和错误，并正常结束线程收尾，不再静默中断。
- 默认日志不写原图、mask/大数组、密钥或输入绝对路径。设置 `TAXAMASK_PREDICTION_DIAGNOSTICS=1` 时才增加候选框摘要。
- 单会话日志默认最多 16 MiB，可用 `TAXAMASK_RUNTIME_LOG_MAX_BYTES` 调整；仍最多保留 20 个会话文件。

## 性能与容量对照

同一 100×80 合成图、64×64 Locator、5 次测量；这是编排开销，不代表真实模型吞吐量。

| 指标 | 拆分前 M1 | 拆分后 M2 |
|---|---:|---:|
| CPU 单样本中位耗时（无内存追踪） | 0.514 ms | 0.810 ms |
| CPU 连续 5 次约计 | 2.85 ms | 4.07 ms |
| Python 峰值（开启 tracemalloc） | 0.092 MiB | 0.184 MiB |
| CUDA 编排中位耗时（开启 tracemalloc） | 0.885 ms | 1.535 ms |
| CUDA 峰值 | 0.094 MiB | 0.094 MiB |
| 每张图默认诊断 | 1 行 / 53 B | 8 行 / 约 2.99 KB |

增加约 0.3 ms 和 0.092 MiB 来自结构化事件对象，未复制模型张量、原图或 mask；结果硬容差全部通过。基线协议对耗时原本标记为 `comparison_only`，因此这里只报告差异，不把它解释为算法变化。
