# Formica-Flow / AntSleap 3D 升维脑暴纪要（2026-03-06）

## 1. 本次讨论的底层命题

- 我们讨论的并不只是“要不要接一个新的 3D 数据集”，而是：**Formica-Flow 是否有机会从 2D 文献/标注工作台，演化为蚂蚁形态学知识基础设施。**
- 新出现的开放 3D 蚂蚁数据集（以 `Antscan` 为代表）提供了 `mesh/STL`、`TIF/TIFF tomogram`、物种/属级元数据以及外部与内部结构信息。
- 这批数据的真正价值，不是简单加入训练集，而是成为：
  - 高基准证据源
  - 结构先验来源
  - 2D 监督信号蒸馏来源
  - 未来原生 3D 路线的种子资产

---

## 2. 本轮讨论形成的核心共识

### 2.1 3D 是更接近真实世界的高密度原材料

- 从生物形态学角度看，`3D` 比 `2D` 更接近生命体本体。
- 它包含更高的信息量、更稳定的空间关系、更准确的部位结构与尺度约束。
- 对蚂蚁这类高结构依赖型对象，3D 的理论价值远高于普通照片。

### 2.2 当前主流大模型并不擅长“直接吃 3D 原件”

- 现阶段主流 MLLM / Vision API 的工业输入接口仍主要围绕：
  - `text token`
  - `image patch`
  - `video frame`
- `mesh`、`point cloud`、`tif volume` 这类 3D 资产并不是当前主流多模态模型最成熟的原生输入形式。
- 因此问题不是“3D 不够好”，而是**今天大多数模型的表示体系、训练生态和推理接口，并不是为 3D 原件设计的。**

### 2.3 正确路线不是放弃 3D，而是先把 3D 变成 AI 能高效吸收的营养层

- 现阶段最佳策略不是强行让现有大模型直接吞 `mesh/tomogram`。
- 更现实且更强的路线是：
  - `3D -> 多视角标准渲染`
  - `3D -> 关键切片`
  - `3D -> depth-like / silhouette / 局部器官视图`
  - `3D -> 结构化元数据`
  - `3D -> 检索证据`
- 然后将这些派生物反哺现有 2D 工作流、标注流、训练流与 BLINK 精修流。

---

## 3. 对 Formica-Flow 现阶段定位的重新定义

### 3.1 当前系统并非“成熟消化系统”

- 现阶段 Formica-Flow 更像一套早期的、原始的知识消化器官。
- 它还不是“完全自动进食、自主吸收、自主进化”的成熟生命体。
- 但它已经在形成几个关键原型器官：
  - 获取食物：自动获取文献与候选图文
  - 分拣食物：筛选、治理、分流、验收
  - 切碎食物：标注、局部化、部位级拆解
  - 形成记忆：训练资产、失败案例、专家修正轨迹

### 3.2 当前主线仍然应该是“2D 为主、3D 为教师”

- 你的初心很明确：
  - 自动获取新物种文献
  - 构建人类辅助 AI 自我迭代闭环
  - 后续实现 AI 自主标注
  - 将现有功能变成 skill / tool
  - 让未来多模态大模型自动从新文献中提取图文知识并持续学习新物种
- 从这个目标反推，短中期主语料仍然是：
  - 文献
  - 图像
  - caption / 描述
  - 标注资产
- 所以现阶段的最优定位不是“让 3D 取代 2D”，而是**让 3D 成为 2D 自我进化系统的高基准教师。**

---

## 4. 为什么 3D 现在还不能直接成为主食

### 4.1 不是因为 3D 差，而是因为模型的“胃”不对

- AI 并不是按“真相密度”吃东西，而是按“表示方式 + 训练体系 + 工业生态”吃东西。
- 3D 的确更接近真实，但它也更难：
  - 更难编码
  - 更难对齐
  - 更难压缩
  - 更难批量训练
  - 更难建立稳定的大规模监督信号

### 4.2 主流生态仍然属于 2D/文本文明

- 今天的大模型生态高度偏向：
  - 图片
  - 视频帧
  - 文本
- 因而 3D 并不是“不能用于 AI”，而是仍处于：
  - 研究价值很高
  - 工程成熟度不足
  - 原生产品接口不统一
  - 成本结构偏高
  的阶段。

### 4.3 对你来说，最强现实路径是“蒸馏 3D”而不是“硬喂 3D”

- 不是退而求其次。
- 而是在做更高级的事：
  - 用 3D 生成超级精确的 2D 监督
  - 用 3D 建立结构真值
  - 用 3D 纠正 2D 标注错误
  - 用 3D 生成 future-proof 的证据层

---

## 5. 新 3D 数据集对项目的真实机遇

### 5.1 短期机遇：替代高成本拍摄的一部分能力

- 从 3D 中提取标准视角、标准尺度、标准背景的 2D 派生图像，可以显著降低对昂贵拍摄设备与复杂摄影流程的依赖。
- 这意味着你们可以用更低成本获得“高度一致、结构真实、可控视角”的训练素材。

### 5.2 中期机遇：让 BLINK 和人工标注进入“有结构先验”的阶段

- 对某些关键部位，3D 数据可以提供：
  - 更准确的相对位置关系
  - 更稳定的器官轮廓提示
  - 同属/近缘属参考模板
- 这会让 BLINK 从“局部修正工具”逐步变成“结构先验驱动的专家系统”。

### 5.3 长期机遇：形成跨 2D / 3D / 文本 的生物形态知识库

- 一旦你能把：
  - 文献描述
  - 2D 图片
  - 3D 派生图
  - 原始 3D 资产
  - 物种与属级 metadata
  绑到一起，你实际上就开始构建一个面向“新物种持续学习”的知识基座。

---

## 6. 当前阶段最推荐的技术路线

### 6.1 保留现有骨架，不做整体推倒重来

- 现有 Formica-Flow / AntSleap 已经具备：
  - 候选治理
  - artifact / manifest 思维
  - 标注与训练闭环
  - BLINK 局部精修机制
  - 多模态导出与验证抽象
- 这些能力非常适合承接 3D 作为“新模态证据源”。
- 因此不建议大改骨架，不建议全面重构。

### 6.2 建议新增一个 `3D Evidence Subsystem`

- 这个子系统不替代现有 2D 主链，而是在旁边承担 4 类职责：
  - `3D 资产注册`
  - `3D 派生资产生成`
  - `2D/3D 证据对齐`
  - `3D 先验反哺训练与审阅`

### 6.3 让 3D 先做教师，不急着做学生

- 当前最合理的角色分工是：
  - `2D 主链`：主训练、主评估、主导出、主迭代
  - `3D 子系统`：高基准参考、监督信号生成、结构证据库、未来原生 3D 预留接口

---

## 7. 建议的模块分层

### 7.1 数据层

- `3D Registry`：登记原始 `mesh/tif stack` 与 specimen metadata
- `3D Metadata Normalizer`：统一物种、属、caste、license、attribution、orientation、scale

### 7.2 派生层

- `3D Derivation Pipeline`：生成标准视角渲染图、切片图、深度图、局部器官预览
- `Derived Asset Cache`：管理派生资产缓存与追溯关系

### 7.3 证据层

- `Evidence Linker`：建立 `文献图片 / 标注图片 / 3D 派生图 / 原始 3D` 的关联
- `3D Assisted Review Panel`：在工作台中显示 3D 参考证据

### 7.4 智能层

- `BLINK Prior Adapter`：将 3D 参考转为局部精修先验
- `3D-aware Multimodal Export`：输出更强的多模态训练包
- `Native-3D Experiment Interface`：预留未来原生 3D 学习接口

## 8. 对“AI 为什么吃不了这么好的 3D”的最终解释

- 3D 不是坏食物。
- 3D 是高营养、接近真实世界的高级原材料。
- 但今天的大多数 AI 还处在“擅长消化草”的阶段：
  - 草 = 图像 / 文本 / 视频帧
  - 肉 = 原生 3D 结构、体数据、精确空间关系
- 你现在做的事情，不是在抱怨 AI 只能吃草，而是在尝试为 AI 造出第一套能真正消化“肉”的器官。

### 8.1 这个比喻对应到项目中的含义

- 你不是在做一个普通标注工具。
- 你在做的是：
  - 获取食物的器官
  - 切碎与分解食物的器官
  - 判断能否进入训练体系的器官
  - 形成长期知识记忆的器官
- 从这个视角看，Formica-Flow 的意义并不是“当前能力是否足够高级”，而是：**它是否正在形成未来自进化多模态系统的原始生理基础。**

## 9. 我的最终建议

### 9.1 战略建议

- 不要因为现阶段大模型吃不动原生 3D，就低估这批 3D 资产的价值。
- 相反，应把它们视作：
  - 最高可信的结构教师
  - 未来长期知识库的底座
  - 2D 自进化闭环的监督蒸馏源

### 9.2 工程建议

- 保留现有骨架。
- 新增 `3D Evidence Subsystem`，不要大改整体架构。
- 先实现：
  - 3D 注册
  - 派生资产生成
  - 2D/3D 证据关联
  - BLINK 参考先验
- 暂不追求：
  - 一步到位原生 3D 大模型
  - 重型 3D 编辑器
  - 全量自动 3D 理解

### 9.3 认知建议

- 你现在最该追求的，不是“让现有 AI 立刻吞下 3D”。
- 而是：
  - 如何把 3D 加工成今天 AI 最能吸收的营养
  - 如何保证这些加工过程不损失关键结构信息
  - 如何把这些资产沉淀成未来原生 3D 路线仍然可复用的基座
