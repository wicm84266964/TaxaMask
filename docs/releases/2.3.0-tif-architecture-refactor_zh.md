## 2026-07-11 TaxaMask 全局第四轮架构优化

本次继续归入 2.3.0 架构整理系列，不创建新版本号或新 Release。继前三轮 TIF 工作台整理后，第四轮把 TaxaMask 主窗口按完整研究工作流拆分，使 2D/STL、Blink、PDF、Agent 和 TIF 集成入口与当前程序状态一致。

### 主窗口与工作流拆分

- `AntSleap/main.py` 从 16,024 行收束到 763 行，MainWindow 类体只保留公开 SAM 信号和 13 行构造器。
- 图片导航/分组、部位树、文献描述、Annotation/SAM、Blink context/workflow、模型管理、训练、预测、VLM、导出、设置/主题分别由独立 workflow owner 负责。
- MainWindow 直接可写状态和直接 Qt 连接均为 0；第五轮完整性恢复接入后，架构扫描的真实连接总数为 197，跨 PDF/TIF/Blink 信号继续由 signal router/coordinator 管理。
- 严格复核后的最大 workflow 模块 1,396 行，避免把拆出的逻辑重新堆成另一个超大文件。

### 数据安全与研究流程

- 图片导入、SAM、parent/child 训练、批量预测、VLM 或 dataset export 运行时阻止切换项目。
- 后台结果验证启动时的 ProjectManager、项目路径和 VLM run id；旧回调不会保存或修改新项目。
- 严格链路复核补齐线程结束与 Qt 回调落地之间的竞态：旧图片导入不继承 provenance，旧 SAM 不写 polygon，旧 parent training error 不触发当前项目重试，旧 Blink 子训练模型不挂到当前项目 route。
- Start Center/Agent 单独记住最近 2D/STL 或 TIF 工作台，最近项目和退出保存不再把页面模式误当项目类型。
- AI prediction/VLM candidate 仍是待复核草稿；manual/confirmed 2D label、Blink trajectory、PDF/STL provenance 和 TIF `manual_truth` 保持独立。
- box-only AI draft 仍不能直接进入训练，必须先通过 SAM 或人工绘制形成 polygon 并由研究者确认。
- 2D SQLite manifest/database、migration、backup/export 和 stale autosave 安全边界保持不变。

### 体验与性能

- PDF 与 TIF 工作台首次进入时创建；同一 2D 图片再次选择时复用 pixmap，文件列表优先局部更新。
- MainWindow 构造中位数由 Stage 0 的 237.394 ms 降至 122.501 ms；Start Center RSS 由 598.883 MB 降至 573.268 MB。
- Start Center 中位数、微秒级图片/部位切换和首次惰性 TIF 创建未达到全部 stretch 目标；release 说明不把行数下降等同于全面性能提升。

### Agent 与验证

- 内嵌 Agent labeling route 已指向 `main_window_agent_context.py`、`main_window_annotation.py`、`main_window_blink_context.py` 和 `main_window_vlm.py`，不再引用 `main.py` 中已移除的方法。
- 完整验证覆盖 18 个 suite、1,149 条测试；1 条环境相关 TIF workbench 测试跳过，其余通过。
- Gate E 修复内嵌 Agent 首问异常压缩和空正文：单次工具结果进入模型上下文前限制为 256 KiB，超大目录列表保留截断与成功状态；无正文、无工具调用的空网关响应自动补发一次简短正文请求。
- 2026-07-12 研究者完成主要交互项验收，暂未发现问题；本结论不等同于 TaxaMask 全功能逐项人工测试，未覆盖的 2D/Blink/VLM/PDF/导出扩展路径继续作为记录在案的人工覆盖缺口。
- 第三、第四轮的结构、信号、跨项目数据安全、性能和验收结论统一整理在 `docs/architecture/taxamask_refactor_round3_round4_summary_zh.md`。

## 2026-07-11 同系列后续维护

第三轮 TIF 工作台优化继续归入 2.3.0 架构整理系列，不新增版本号或 Release。相关代码已进入 `main` 分支；`v2.3.0` 标签及其自动生成的源码压缩包仍是 2026-07-09 的发布快照，需要第三轮改动时请使用最新 `main`。

本轮按完整研究工作流继续拆分 Selection/Lifecycle、Annotation/Save/Truth、ROI-to-Part、Part Mask/Material、Volume Preview、Local Axis/Reslice、Backend/Result Review，以及 Shell/View/Signal Router/Coordinator。以发布基线统一口径计算，`AntSleap/ui/tif_workbench.py` 从 14,234 行收束到 5,041 行，当前保留 219 个 Widget 方法和 11 处直接 Qt 信号连接。主文件已经显著缩小，但 controller 仍持有完整 Widget，部分 controller 仍较大，因此不宣称已经完全解耦。

### 本轮实际体验修复

- 修复部位绑定标签表在重新进入程序后没有恢复的问题；现在以该部位保存的 `training.label_schema_id` 为准。
- 修复三维体预览仍调用拆分前 Widget 回调导致的崩溃；GPU 画布渲染状态和鼠标旋转、平移、缩放统一对接 `TifVolumeRenderController`。
- 优化 specimen、part 和 reslice 切换：选择回调只做当前切片所需工作，三维重绘合并到选择完成后的后续事件。
- 大型已保存重切片改为后台准备 preview 后再上传 GPU，旧 image/label memmap 等待旧预览结束后在后台释放，减少短暂无响应并避免提前关闭仍被后台使用的数组。
- 严格 label-ID 全量扫描不再发生在 specimen/part/reslice 选择过程中，只在接受 `manual_truth` 或训练就绪核验时执行。

### Agent 与验收对齐

- “询问 Agent”新增紧凑的体数据生命周期摘要，可区分选择加载、后台 preview、选择后待刷新和旧数组后台释放，避免把正常后台准备误判为界面卡死。

## 版本定位

TaxaMask 2.3.0 是一个重要的 TIF/CT 工作台架构版本。这个版本的目标不是改变研究者每天使用的入口，而是把已经可用但持续膨胀的 TIF 工作台整理成更容易维护、测试和扩展的专业软件结构。

本版本继续保持以下研究数据原则：

- 原始 TIF/CT stack 不被覆盖。
- AMIRA 原始文件只读兼容，不写回原始数据。
- AI 预测结果默认进入可复核草稿，不会自动成为 `manual_truth`。
- 训练样本只使用人工复核后的训练真值。
- Local Axis reslice 是 part 下的派生输出，不修改 source stack。
- 后台任务完成前，保存、训练、预测、导入、ROI 创建和预览重建不会互相踩状态。

## 主要变化

### TIF 工作台职责拆分

- 拆分 TIF 工作台中的翻译表、弹窗、画布、worker、helper、style、layout、page 和 control panel。
- 保留 `TifWorkbenchWidget` 作为外部入口，但让新业务逻辑优先进入 core/service/task/controller 层。
- 右侧栏、页面栈和控件 objectName 增加测试保护，降低 UI 拆分后按钮消失、滚动失效或翻译漏出的风险。

### 数据安全规则集中化

- 新增或强化 label role 写入矩阵。
- 强化 `manual_truth` promotion policy。
- 强化 prediction import policy。
- 增加 write intent guard，避免写入越界或覆盖 source stack。
- 保持 `editable_ai_result`、`raw_ai_prediction_backup` 和 `manual_truth` 的职责边界。

### Service / Controller 层

新增或整理以下 TIF service/controller：

- selection state。
- label edit request。
- truth promotion。
- ROI part request。
- backend workflow selection。
- volume preview request。
- training / prediction panel controller。
- preview / resource controller。
- Local Axis draft/frame/reslice controller。
- Agent context builder。

这些模块让训练、预测、预览、重切片和保存逻辑更容易单独测试，也为后续 WebUI、CLI、批处理和数据治理打基础。

### 统一 task/state 生命周期

- 新增 task context / task state / task manager。
- 将高风险后台链路纳入统一任务管理：
  - TIF / AMIRA import。
  - materialize。
  - label auto/manual save。
  - truth promotion。
  - ROI confirm。
  - mask preview。
  - volume preview。
  - backend prepare/train/predict。
  - Local Axis export。
- 增加 selection/edit/preview/backend/ROI/local-axis 状态摘要。
- 旧后台任务回调需要匹配当前 specimen / part / reslice，减少刷新错误界面的风险。

### 3D Preview 与资源提示

- 增加 TIF preview/resource controller。
- 对系统内存、Windows 提交内存/页面文件、GPU/OpenGL、volume IO 等失败做统一分类。
- 资源不足时倾向于显示可理解提示，避免研究者误以为项目数据损坏。
- 切片查看和标注链路尽量不因 3D preview 不可用而中断。

### Local Axis Reslice

- 拆出 Local Axis UI controller。
- 保留三点参考、观察侧剖切面、roll reference、对齐和 reslice payload 准备链路。
- 保留关键规则：预览类 busy lock 不阻止合法的 Local Axis 三点草稿交互。
- Local Axis 输出仍作为 part 下的派生 reslice 保存，不修改 source stack。

### Training / Prediction

- 拆出训练与预测面板控制逻辑。
- 训练样本选择继续优先使用 train-ready part/reslice manual truth。
- 预测目标支持 current part、all ready 和 group tag 筛选。
- 预测导入继续进入可复核结果层，不直接覆盖 `manual_truth`。
- 训练结果、模型 manifest 和模型库记录的 UI 状态更集中。

### Agent / AntCode 上下文

- TIF Ask Agent 上下文新增并保留关键现场字段：
  - 当前 specimen / part / reslice。
  - 当前 label role、label schema 和 selected material。
  - 训练样本状态和预测目标摘要。
  - backend run 状态。
  - preview/resource 状态。
  - Local Axis 状态。
  - task/state summary。
- 主窗口压缩上下文和 Agent 面板展示已对齐，避免生成字段在转交给 Agent 前被丢掉。
- Agent route 的源码引用更新到新 controller/service 文件。

### UI 与体验修复

- 修复 TIF 标签表和画笔标签选择重复的问题，将绑定到当前 part 的彩色标签表作为画笔选择入口。
- 增加从无标签重切片切换到有标签切片时的加载反馈。
- 修复 PDF 处理链路浅色模式下的黑色按钮问题。
- 修复训练与预测区域部分英文漏翻译和右侧栏溢出回归。
- 增强新建 label schema / part group tag 时的操作反馈。

## 研究流程收益

- 后续新增 TIF 功能时，不需要继续把逻辑堆进一个巨型 PySide Widget。
- 保存、训练、预测、导入、预览和重切片的状态更容易定位。
- 预测导入、raw backup、manual truth promotion 和训练样本选择有集中 guard 和测试。
- Local Axis 三点、重切片和保存重开链路有更清晰的控制边界。
- Ask Agent 能更准确判断当前卡在保存、训练、预测、预览、资源不足还是 Local Axis。
- 当前版本为后续只读 project health check、完整项目备份、manifest/sidecar 一致性检查和数据治理打基础。

## 发布前人工验收

本版本发布前已使用真实 TIF 项目完成基础人工验收，未发现阻断发布的问题。人工确认覆盖：

1. 打开真实 TIF 项目。
2. 切换 specimen、part、reslice。
3. 查看切片和标签层。
4. 保存 working edit 并重开确认。
5. 打开 3D volume preview。
6. 检查 mask / boundary / masked image 显示。
7. 走完整 Local Axis 三点参考、对齐、重切片、保存重开链路。
8. 检查训练/预测入口、预测分组和目标选择。
9. 导入预测结果，确认进入 editable AI result / review draft，而不是直接覆盖 `manual_truth`。
10. 在资源紧张时确认提示可理解，项目数据未被修改。

后续在更多机器、更大体积 TIF/CT stack 和不同显卡环境中使用时，仍建议把这些路径作为日常回归检查重点。
