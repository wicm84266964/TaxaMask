# Formica-Flow / AntSleap 发展讨论总结（2026-03-05）

## 1. 本次讨论核心结论

### 1.1 质量红线
- 自动化标注目标明确为：**目标视角准确率 >= 95%**。
- 如果某方案无法稳定达到该标准，则判定为当前阶段不胜任。

### 1.2 级联专家策略
- 级联专家由解剖学专家（你）主导定义，不由系统自动假设关系。
- 在你完成部位级联关系定义前，级联专家推理保持**默认关闭/受门禁约束**。

### 1.3 模型路线认知
- “多模型”本身不是问题，**无治理的模型堆叠**才是问题。
- 后续必须采用“准入-路由-淘汰”治理策略，避免系统臃肿。

---

## 2. 当前程序状态（与讨论同步后的认知）

### 2.1 已实现并可用
- BLINK 框架基础链路已打通：轨迹数据、训练、推理协议、UI 线程化、级联门禁。
- 推理输出已结构化，单图与批量写回逻辑已统一。

### 2.2 当前限制（讨论中反复确认）
- 项目训练标签中尚无原生 `view` 字段，训练集不具备视角级强约束。
- 默认训练切分仍是随机切分，不是视角分层切分。
- PDF 文献筛选/提取流程尚未正式对接到标注工作台的“强规则数据入口”。

---

## 3. 暂定优化方向（下一阶段）

> 该方向为本次会话达成的“临时锁定方案”，后续可按实验结果再修正。

### 3.1 宏观定位器训练范围
- 暂定先聚焦：**侧视图 + 俯视图**（一个定位器模型）。
- `head-view` 先作为筛选分流数据，不并入当前主训练流。

### 3.2 泛化增强策略
- 采用**保守方向增强**：小角度旋转、轻量尺度/亮度扰动。
- 避免会破坏解剖语义的一步到位强增强（大角度、无约束翻转等）。

### 3.3 数据入口策略
- 不建议“全类型图片一锅训练”（三视图 + 电镜 + 极端局部特写）。
- 先建立分流：
  - `Core-2`（侧视+俯视，主生产训练集）
  - `Frontier`（电镜/局部特写，研究队列）
  - `Ambiguous`（待复核队列）

---

## 4. 下一步工作包（讨论后建议）

### WP-A：视角过滤与数据治理（优先级最高）
- 建立可审计的视角筛选规则（至少能稳定排除 `head-view` 与非目标图）。
- 形成“进入训练集”的可追踪标准（日志 + 阈值 + 人工复核比例）。

### WP-B：单模型可行性验证（侧视+俯视）
- 以 `>=95%` 为硬验收，按视角分别评估，不做总平均掩盖。
- 明确失败判据：任一目标视角低于阈值即不通过。

### WP-C：增强策略逐步上线
- 在不破坏标签语义前提下逐步加保守增强。
- 每轮增强都要有“是否提升目标视角精度”的对照记录。

### WP-D：PDF 到标注台的正式桥接
- 将文献筛选/提取输出与标注工作台导入策略统一。
- 杜绝“提取端通过但训练端污染”的链路断裂。

---

## 5. 里程碑式验收建议

### M1（数据入口稳定）
- 训练输入集可明确区分 `Core-2 / Frontier / Ambiguous`。
- 非目标图混入率显著下降且可追踪。

### M2（单模型精度验证）
- 侧视与俯视分别达到或接近 95% 目标。
- 若未达标，进入“分视图模型”或“更强定位器”备选路线评估。

### M3（流程闭环）
- 文献筛选 -> 提取 -> 导入 -> 标注 -> 训练 -> 回写 全链路规则一致。
