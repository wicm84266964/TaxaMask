## 1. 实施原则

这次改造的核心不是“多加几个输入框”，而是把 PDF 后半段工作流拆清楚：

- PDF Screening Profile：决定哪些 PDF 文献进入后续处理。
- Figure Extraction / Review Profile：决定怎样提取图版、怎样组装图文证据、启用多模态时怎样复核图版。
- Text LLM API：服务 PDF 文本筛选。
- Multimodal LLM API：服务图像 + 文本的多模态复核。

实施时必须保证：

- 旧的蚂蚁三视图流程默认可继续运行。
- 不启用多模态验证时，Figure Extraction / Review Profile 仍然影响图文提取阶段。
- 多模态 API key、文本 API key 不写入开源 profile。
- mock/default 复核不能冒充真实多模态验证。

3. 阶段 C：让图文提取吃 profile。
