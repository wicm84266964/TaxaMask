日期：2026-06-20

```text

```

## 执行目标

本清单用于指导 `pca_head_orientation` 原型能力合并进 TaxaMask 的第一版实现。

第一版不是单样本手工试验工具，而是面向 AntScan 2193 个个体数据的成熟产品骨架。必须同时覆盖：

```text
手工可靠标定
训练素材结构化保存
AI proposal 接入
批量复核队列
accepted 后确定性重切片导出
```

模型效果可以迭代，但数据结构、状态流转、复核交互和导出链路必须从第一版开始稳定。

## 总体原则

- 不搬运独立 PySide6 工作台。
- 不继续使用 `pca_head_orientation` 的项目 JSON 作为 TaxaMask 主模型。
- 不把功能命名为 Brain Reslice。
- 不把脑部逻辑写死。
- 不把体信号作为主定位算法。
- 不迁移 QC 图作为第一版正式界面。
- 不覆盖原始 TIFF、working volume 或 part image。
- 不让 AI proposal 未经 accepted / human confirmed 就静默导出正式结果。
- 不为 Local Axis AI 另起一套孤立模型接入方式；必须复用 TaxaMask 现有 contract、manifest、run 目录、日志、model profile 和 reviewable draft/proposal 思路。
- 灰度 image 重切片使用 linear interpolation。
- mask / label 重切片使用 nearest-neighbor。
- 所有正式输出挂在 `specimen -> part -> reslices` 下。
- 所有人工确认和 AI proposal 都要可追溯、可复核、可训练。

## Phase 2：Local Axis Reslice 核心几何与导出

目标：实现不依赖 UI 的核心功能。给定 part volume、origin、output axis、roll reference，能计算 local frame、生成 preview、导出 TIFF 和 metadata。

新增文件：

```text

tests/test_tif_local_axis_reslice.py
```

核心数据结构：

```text

roll_reference
local_frame
reslice_params

```

核心函数：

```text
source_z_axis_for_part(part_shape_zyx)
create_editable_axis_from_source(source_axis)
normalize_vector_zyx(vector, spacing_zyx=None)
compute_local_frame(origin, output_axis, roll_reference, spacing_zyx)
validate_local_frame(local_frame)
reslice_volume(volume, local_frame, params, interpolation)
build_reslice_preview(volume, local_frame, params)
write_reslice_tiff(path, volume)
write_reslice_metadata(path, payload)
export_part_reslice(project, specimen_id, part_id, payload)
```

几何规则：

- [ ] source Z axis 默认从 part 的 `z=0` 到 `z=z_last`，穿过 part 中心。
- [ ] editable output axis 可由 source Z axis 复制。
- [ ] output Z axis 表示输出切片序列推进轴。
- [ ] roll reference point pair 决定绕 output Z axis 的旋转。
- [ ] local frame 必须包含 `origin_zyx`、`x_axis_zyx`、`y_axis_zyx`、`z_axis_zyx`、`output_axis`。
- [ ] axis 计算应考虑 spacing，避免 anisotropic voxel 造成方向偏差。
- [ ] output axis 和 roll reference 退化时必须报错或返回 validation error。

重采样规则：

- [ ] image interpolation 默认 `linear`。
- [ ] mask interpolation 必须 `nearest`。
- [ ] 默认 output spacing 继承 part spacing。
- [ ] isotropic spacing 只作为高级选项，不作为默认。
- [ ] preview 低分辨率临时生成，第一版不必须保存。

输出路径：

```text
specimens/<specimen_id>/parts/<part_id>/reslices/<reslice_id>/
  image.tif
  metadata.json
  mask.tif        optional
```

metadata 必须包含：

- [ ] schema_version。
- [ ] reslice_id。
- [ ] template_id。
- [ ] human_confirmed。
- [ ] source specimen / part / part image / part mask。
- [ ] parent_bbox_zyx。
- [ ] spacing_zyx。
- [ ] source_axis。
- [ ] editable_axis。
- [ ] origin。
- [ ] roll_reference。
- [ ] local_frame。
- [ ] reslice_params。
- [ ] outputs。
- [ ] quality。
- [ ] model/proposal provenance，如果来自 AI proposal。

测试：

- [ ] source Z axis 生成正确。
- [ ] output axis + roll pair 能生成正交 local frame。
- [ ] roll pair 与 output axis 平行时返回 invalid。
- [ ] linear image reslice shape 正确。
- [ ] nearest mask reslice 不产生新 label ID。
- [ ] export 不覆盖 part image。
- [ ] export 不覆盖 part mask。
- [ ] metadata path 和 project record 一致。

验收标准：

- [ ] 无 UI 情况下可用单元测试完成一次 part reslice export。
- [ ] 原始 TIFF、working volume、part image 均不被修改。

## Phase 3：统一模型后端契约、训练素材与 AI proposal

## 推荐执行顺序

建议严格按以下顺序实施：

```text
0. 基线检查
1. tif_project helper
2. tif_local_axis_reslice core
3. tif_local_axis_ai contract / manifest / proposal import
4. tif_local_axis_batch queue / run
5. Local Axis Reslice 单样本页面
6. TIF workbench 左侧树和入口
7. Review queue / Model panel
8. baseline / external proposal import
9. 端到端验收
10. 再考虑 Source Z Signal 辅助诊断
```

不要先做复杂 3D gizmo，也不要先做 QC 图迁移。第一优先级是数据结构、可训练 metadata、proposal 状态、accepted 导出和批量复核闭环。
