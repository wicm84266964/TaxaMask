# TaxaMask 1.3.0 TIF Developer Preview

Release date: 2026-06-29

Branch: `codex/antscan-stl-tif-rearchitecture`

Tag: `v1.3.0-tif-preview`

This release upgrades the TIF/CT developer-preview branch with the new GPU-streamed 3D preview path from PR #1. It keeps the 2D SQLite storage work from TaxaMask 1.2.0 and the TIF SQLite project-index route from `v1.2.0-tif-preview`.

## Research Workflow Impact

The main goal of this release is smoother and more trustworthy 3D inspection of large TIF/CT stacks on ordinary research hardware.

For ant CT work, the 3D view is often used to decide whether a candidate ROI, part mask, or local-axis frame is worth deeper slice review. This release makes that preview path closer to professional volume-viewer behavior while keeping it read-only: it helps inspection, but it does not silently edit TIF stacks, masks, parts, or training data.

## TIF / CT 3D Preview

- Adds streamed GPU preview building for large TIF volumes.
- Adds GPU texture caching so repeated specimen, ROI, and mask previews can reuse uploaded textures.
- Adds ROI high-detail inspection for local structures.
- Adds transfer-function presets for morphology-oriented volume viewing.
- Adds separate drag and still preview quality paths:
  - drag uses lighter preview settings for interaction
  - still view rebuilds a sharper preview after the view settles
- Adds GPU budget reporting and automatic preview downscaling when a requested texture would exceed the budget.

For researchers, this means large TIF scans should feel less like static image stacks and more like inspectable 3D specimens, while still preserving the slice-review route for precise annotation.

## TIF Import And Materialization

- Adds metadata-only TIF registration.
- Adds explicit materialization from a registered TIF into the working volume sidecar.
- Keeps source TIFF stacks outside the project database.
- Keeps generated working images, labels, part volumes, masks, and reslice outputs as sidecar files.

For large scans, metadata-only registration lets the project record the specimen before committing time and disk I/O to full sidecar generation.

## Preview Safety

- 3D preview remains read-only.
- ROI high-detail inspection reads from the source bbox and does not write source data.
- Part mask preview and masked-image modes are display aids, not automatic label acceptance.
- Precise edits still belong in slice review.

This matters because 3D preview is excellent for orientation and structure review, but training-quality labels still need auditable acceptance steps.

## Windows Startup

The PR was merged into the TIF branch, then the Windows launcher was restored to the branch's broader environment-discovery behavior. The launcher keeps support for custom `taxamask*` and `antsleap*` conda environments instead of only checking a narrow fixed environment name.

## Data Boundary

This release does not include private research data.

The repository intentionally excludes local project JSON files, SQLite databases, TIF/CT stacks, generated sidecars, model weights, API settings, and private design notes.

## Known Limitations

- TIF/CT remains a developer preview, validated mainly with AntScan ant CT data.
- 3D preview is for inspection and orientation; slice review remains the precise editing path.
- Actual GPU behavior should still be checked on target machines, especially laptops with shared thermal and memory constraints.
- Metadata-only TIF registration requires a later materialization step before full 3D preview can be built.
